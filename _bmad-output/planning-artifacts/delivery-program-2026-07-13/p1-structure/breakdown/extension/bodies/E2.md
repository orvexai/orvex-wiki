## 🎯 Feature area

For a non-technical user, "connect your AI" must be one guided step — no MCP config, no local server, no JSON (FR-D2). This epic owns the identity handshake between the extension and Orvex: a guided connect flow that mints a **delegated principal**, a **scoped** token held in secure extension storage (never a leakable secret), and an honest per-assistant connection-status surface. It is the trust spine that every delivery and consent story leans on — a delivery adapter may only act as a principal this epic minted and scoped.

**🧾 Code audit (2026-07-14):** absent — repo is a pre-definition scaffold (README only). No auth flow, no token storage, no status surface exist; the family's recurring "trusted-header / deferred-auth" defect is closed here at the client edge. [Source: g9vWbSYplh FR-D2, FR-D4, FR-O4]
