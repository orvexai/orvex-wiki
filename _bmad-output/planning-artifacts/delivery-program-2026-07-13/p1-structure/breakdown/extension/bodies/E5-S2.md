## 🎯 Story

As a **maintainer** I want **versioned per-provider selector fixtures plus a canary heartbeat that self-detects when a provider's web UI has changed shape** so that **the extension notices its own breakage the moment a provider moves the compose surface, instead of silently injecting into the wrong place or nothing (FR-D7)**.

**Definition of Done:** one named test `TestCanaryDetectsProviderUIBreakage` (integration — against a mutated provider DOM fixture the canary flips to "broken" within one heartbeat/delivery attempt, identifies which provider and which selector failed, and marks that provider degraded; against the healthy fixture it stays green). *Exact heartbeat cadence + fixture schema are pinned at ENG-2690; this story is dispatch-blocked until that contract exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a provider whose compose selector no longer resolves, When the canary runs, Then it reports that provider broken with the failing selector named. *Assert: canary state == broken(provider, selector).* [Source: g9vWbSYplh FR-D7 (self-detect breakage)]
- [ ] **AC2** — Given a healthy provider UI, When the canary runs, Then it stays green with no false positive. *Assert: unchanged fixture → green.* [Source: g9vWbSYplh FR-D7]
- [ ] **AC3** — Given selector fixtures, When the provider ships a new UI version, Then fixtures are versioned so a repair is a data change, not a code rewrite. *Assert: fixture bump alone repairs resolution.* [Source: g9vWbSYplh FR-D7 (durable maintenance)]
- [ ] **AC4 (no-silent)** — Given breakage detected, When delivery is attempted, Then injection is suppressed and the E5-S1 fallback + E5-S3 alert fire — never a blind write. *Assert: broken → no injection, fallback+alert triggered.* [Source: g9vWbSYplh FR-D7, FR-D5]
