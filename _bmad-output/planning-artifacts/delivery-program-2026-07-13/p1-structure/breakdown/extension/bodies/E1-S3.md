## 🎯 Story

As a **security owner** I want **the manifest to request least-privilege host permissions, a hardened CSP, and content scripts that read only the compose surface — no broad page capture** so that **the extension touches third-party pages with the minimum blast radius and cannot become a data-exfiltration vector (FR-D5)**.

**Definition of Done:** one named test `TestPermissionsAreMinimalAndNoBroadCapture` (integration — the built manifest declares only the beachhead host origins and no `<all_urls>`/`tabs`/`history`; a static scan proves content scripts query only the designated compose selectors, never `document.body`/full-page text). *Exact permitted origin set + CSP string are pinned at ENG-2690; this story is dispatch-blocked until that contract exists.*

- [ ] **AC1** — Given the built manifest, When permissions are audited, Then `host_permissions` lists only the beachhead provider origins and no `<all_urls>`, and no `tabs`/`history`/`cookies` scopes appear. *Assert: manifest permission diff against the allowlist is empty.* [Source: g9vWbSYplh FR-D5 (no scrape, own surface only)]
- [ ] **AC2** — Given a content script, When it runs on a provider page, Then it reads only the compose-surface node(s), never whole-page DOM/text. *Assert: static scan finds no `body.innerText`/`document.documentElement` reads.* [Source: g9vWbSYplh FR-D5; NFR-2]
- [ ] **AC3** — Given the extension pages, When CSP is evaluated, Then no remote script/style/connect is allowed beyond the Orvex API origin. *Assert: CSP forbids `unsafe-eval` and non-Orvex `connect-src`.* [Source: g9vWbSYplh FR-S6 spirit (untrusted third-party page)]
