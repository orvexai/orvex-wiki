## 🎯 Story

As a **non-technical user** I want **to connect my assistant to Orvex in one guided step — no MCP config, no local server, no JSON — that mints a delegated principal** so that **the extension can deliver on my behalf and I reach a first enriched output without touching a config file (FR-D2, FR-O4)**.

**Definition of Done:** one named test `TestGuidedConnectMintsDelegatedPrincipalWithinBudget` (integration — a scripted first-run walkthrough completes connection in ≤3 discrete taps, mints a delegated principal bound to the user, and requires zero manual config input; a config-file/CLI step anywhere fails the test). *Exact delegated-principal handshake + scope claims are pinned at ENG-2690; this story is dispatch-blocked until that contract exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a fresh install, When the user runs Connect, Then a delegated principal is minted and bound to the user with no JSON/MCP/CLI step. *Assert: principal id issued; zero free-text config fields presented.* [Source: g9vWbSYplh FR-D2]
- [ ] **AC2** — Given the guided flow, When measured, Then it completes in ≤3 taps and ≤2 minutes. *Assert: harness tap-count ≤3, wall-clock ≤120 s.* [Source: g9vWbSYplh FR-O4]
- [ ] **AC3** — Given a user who cancels mid-flow, When they retry, Then no partial principal persists and the flow restarts cleanly. *Assert: no orphaned token in storage.* [Source: g9vWbSYplh FR-D4 (never a half-authorized state)]
- [ ] **AC4 (error-path)** — Given the Orvex API is unreachable, When Connect runs, Then it fails loud with a retry affordance, never a fake-connected state. *Assert: status = error, not connected.* [Source: g9vWbSYplh NFR-4]
