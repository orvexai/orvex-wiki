## 🎯 Story

As a **product owner** I want **a moderated first-run usability test that verifies a non-technical participant connects an assistant in ≤3 taps and ≤2 minutes with no config file or command line** so that **FR-O4 — the concrete acceptance test for one-step connect (FR-D2) — is proven with real users, not just asserted in code (FR-O4, FR-D2)**.

**Definition of Done:** one named artifact/gate `FR-O4-usability-evidence` (a moderated first-run usability protocol run with ≥5 non-technical participants: each completes connection in ≤3 taps and ≤2 minutes with zero config/CLI; results recorded with tap-count, time-to-connect, and completion rate; a pass requires the target met by the pre-agreed majority). *Exact participant profile + pass threshold are pinned at ENG-2690; this story is dispatch-blocked until that contract exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given non-technical participants, When each runs first-run connect, Then the median completes in ≤3 taps and ≤2 minutes. *Assert: median tap-count ≤3, median time ≤120 s.* [Source: g9vWbSYplh FR-O4]
- [ ] **AC2** — Given the flow, When observed, Then no participant is required to touch a config file, JSON, or command line. *Assert: zero config/CLI steps in every session.* [Source: g9vWbSYplh FR-O4, FR-D2 (no MCP config, no JSON)]
- [ ] **AC3** — Given the sessions, When aggregated, Then the completion rate meets the pre-agreed pass threshold and blockers are logged. *Assert: completion rate ≥ threshold; blocker list captured.* [Source: g9vWbSYplh FR-O4 (verified by first-run usability testing)]
- [ ] **AC4 (regression tie-in)** — Given the automated E2-S1 tap-budget test, When it and the usability evidence disagree, Then the discrepancy is escalated (the human test is authoritative for FR-O4). *Assert: mismatch → escalation, not a silent pass.* [Source: g9vWbSYplh FR-O4]
