## 🎯 Story

As a **release engineer** I want **signed, review-passing packages for the Chrome Web Store and Firefox AMO, with semantic versioning and a validated auto-update channel** so that **a non-technical user can install and stay current on the write/compose client (AD-5a) through the official stores**.

**Definition of Done:** one named test/gate `TestStorePackagesPassReviewLintAndAutoUpdate` (CI — the produced Chrome + Firefox packages pass store-review-constraint lint (permission justification, no remote code, CSP), carry a bumped semver, and resolve against the auto-update manifest). *Exact store listing metadata + update-channel URLs are pinned at ENG-2690; this story is dispatch-blocked until that contract exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given the built extension, When packaged, Then Chrome Web Store + Firefox AMO review-lint pass (no remote code, justified minimal permissions, compliant CSP). *Assert: `web-ext lint` + CWS policy checks clean.* [Source: AD-5a; g9vWbSYplh FR-D5 (minimal permissions survive store review)]
- [ ] **AC2** — Given a release, When published, Then it carries a monotonically bumped semantic version. *Assert: manifest version > prior; tag matches.* [Source: AD-5a]
- [ ] **AC3** — Given an installed prior version, When a new version publishes, Then the auto-update channel resolves and upgrades cleanly. *Assert: update manifest resolves; no manual reinstall.* [Source: AD-5a; NFR-4]
- [ ] **AC4 (negative)** — Given a package that adds an unjustified permission or remote script, When review-lint runs, Then it fails the release. *Assert: planted violation → RED, no publish.* [Source: g9vWbSYplh FR-D5; family permissions doctrine]
