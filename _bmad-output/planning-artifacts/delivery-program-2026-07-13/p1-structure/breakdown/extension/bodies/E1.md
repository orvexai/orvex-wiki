## 🎯 Feature area

`orvex-studio-extension` is the family's 17th component and its **write/compose client port** (AD-5a) — the surface that carries Memory-enriched prompts into third-party AI web UIs. It is NOT the MCP (AD-5b): MCP reads/serves; the extension writes into a page the user already controls. Nothing in F1 can be built until the client substrate exists: an MV3 (Manifest V3) extension with a service worker, a popup shell, env-only config, a build+CI pipeline, a generated TS contracts client, an AGPL-import guard (the engine is AGPL; satellites/clients must not link it), and a least-privilege permissions floor. This epic is the load-bearing foundation every later epic stands on.

**🧾 Code audit (2026-07-14):** absent — repo is a pre-definition scaffold (README only). The whole component is 100% pre-build; every recurring family defect (deferred auth seam, over-broad permissions, unbuilt shared client) is front-loaded here so no later story re-invents it.
