## 🎯 Feature area

Provider web UIs change without notice; a delivery mechanism that silently injects into the wrong place — or nothing — is worse than none. This epic makes breakage **loud and safe**: a copy/paste fallback that delivers the prompt when a provider blocks injection (FR-D5), versioned selector fixtures plus a canary heartbeat that self-detects when a provider's UI has moved (FR-D7), and a fail-loud UX that notifies the user, auto-falls-back to copy/paste, and alerts the team (FR-D7). Never silent injection into the wrong surface, never silent nothing.

**🧾 Code audit (2026-07-14):** absent — repo is a pre-definition scaffold (README only). No fallback, fixtures, canary, or alerting exist. This epic operationalizes the family "no fallbacks that silently rot" doctrine as *loud* fallback — a fallback the user and team both see. [Source: g9vWbSYplh FR-D5, FR-D7; NFR-4]
