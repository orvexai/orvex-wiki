## 🎯 Story

As a **user and as the delivery team** I want **breakage to surface as a clear in-product message with an automatic copy/paste fallback, and to raise a team alert** so that **when a provider changes its UI the user is told and kept working, and the team is paged to ship a fixture fix — never a silent degradation (FR-D7)**.

**Definition of Done:** one named test `TestBreakageFailsLoudFallsBackAndAlertsTeam` (integration — when the E5-S2 canary reports a provider broken, the popup shows a clear breakage state, delivery auto-routes to the E5-S1 copy/paste fallback, and a redacted team alert is emitted exactly once per breakage episode). *Exact alert channel/payload contract is pinned at ENG-2690; this story is dispatch-blocked until that contract exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a canary-reported breakage, When the user next delivers, Then the popup shows a clear, honest "delivery to <assistant> is temporarily broken — here's your prompt to paste" state. *Assert: breakage UX rendered; not a generic error.* [Source: g9vWbSYplh FR-D7 (notify the user)]
- [ ] **AC2** — Given breakage, When delivery runs, Then it auto-falls-back to E5-S1 copy/paste with no user configuration. *Assert: fallback engaged automatically.* [Source: g9vWbSYplh FR-D7 (fall back to copy/paste)]
- [ ] **AC3** — Given breakage, When detected, Then a team alert fires — deduplicated to once per episode — carrying provider + selector + version, PII-redacted (no memory/page content). *Assert: one alert per episode; payload redacted per E4-S3.* [Source: g9vWbSYplh FR-D7 (alert the team); NFR-6]
- [ ] **AC4 (recovery)** — Given the provider UI (or a fixture bump) restores resolution, When the canary greens, Then the breakage UX clears and normal injection resumes. *Assert: green → UX cleared, injection resumes.* [Source: g9vWbSYplh FR-D7]
