## 🎯 Story

As a **user** I want **a per-assistant connection-status surface that tells me plainly which assistants are connected, which are supported, and which are honestly not-yet-supported** so that **I always know where delivery will work and I am never misled into thinking an unsupported assistant is wired up (FR-D2, FR-D3)**.

**Definition of Done:** one named test `TestStatusSurfaceReflectsPerAssistantTruth` (integration — the popup shows connected/disconnected per beachhead assistant and an explicit "not yet supported" state for out-of-beachhead assistants; a disconnected or unsupported assistant is never rendered as ready). *Exact assistant registry + status vocabulary are pinned at ENG-2690; this story is dispatch-blocked until that contract exists.*

- [ ] **AC2** — Given an assistant outside the beachhead, When shown, Then it displays an honest "not yet supported" state, not a fake-connectable one. *Assert: unsupported → non-actionable "not yet supported" label.* [Source: g9vWbSYplh FR-D3 (honest not-yet-supported messaging)]
