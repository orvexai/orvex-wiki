## 🎯 Story

As a **privacy-conscious user** I want **a per-use consent gate that stops any private memory from entering a third-party prompt until I explicitly opt in for that use** so that **my private memories are never silently injected into ChatGPT/Gemini (FR-D4)**.

**Definition of Done:** one named test `TestPrivateMemoryRequiresPerUseConsentBeforeInjection` (integration — a composed prompt containing a private-flagged memory is blocked at the gate until a per-use consent is recorded; without consent the injection call is never reached; the consent is scoped to that single delivery, not a standing grant). *Exact sensitivity fields + consent record schema are pinned at ENG-2690; this story is dispatch-blocked until that contract exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a composed prompt with a private-flagged memory, When delivery is requested, Then the gate blocks injection and prompts for per-use consent. *Assert: E3 injection port not called until consent recorded.* [Source: g9vWbSYplh FR-D4 (per-use consent, never silent injection)]
- [ ] **AC2** — Given the user grants consent, When they confirm, Then injection proceeds and a consent record (memory, provider, timestamp, single-use) is written. *Assert: audit record persisted; scope == single delivery.* [Source: g9vWbSYplh FR-D4; NFR-6 (auditability)]
- [ ] **AC3** — Given the user declines, When they cancel, Then the private memory is excluded and delivery either proceeds without it or aborts per the user's choice — never injected. *Assert: declined memory absent from injected payload.* [Source: g9vWbSYplh FR-D4]
- [ ] **AC4 (negative)** — Given a prior consent for an earlier delivery, When a new delivery includes the same private memory, Then consent is requested again (no standing grant leakage). *Assert: second use → fresh consent prompt.* [Source: g9vWbSYplh FR-D4 (per-*use*)]
