## 🗺️ Epic — Memory subsystem (FormSpec, privacy, lifecycle)

Memory is the curated-facts layer of the three-layer context system — "your RAM," the flagship legible portrait of what Orvex knows about you. orvex-studio-api is the **system-of-record** for the product Memory card (`/v1/memory`, AD-1a). This epic owns: the AI-declared **FormSpec** (structure only — the AI never authors a fact, ADR-0001/NFR-SA4), user-filled values over the fixed 9-type palette, the frozen **versioned card schema** (FR-M1–4 / AD-2), the **3-state per-answer privacy** model (Open / Private / Shared-private) and its `consent_events` enforcement pipeline (FR-SA6), the privacy **contract** propagated into knowledge's Memory corpus (FR-SA7 — Private/Shared-private never in default retrieval, never over MCP), the **bi-temporal lifecycle + Reconcile** state machine (FR-L1–5 / AD-3), the **confirm-gate-as-sole-mutation-path** discipline (AD-4), the deterministic **FR-31 weave** (Free path, FR-SA8), and the **team-aware-but-not-team-functional** schema (FR-T1–4 / AD-12).

This epic exists because privacy is the single most-tested surface (SM-C3 = 0 leaks, NFR-SA2) and because "your AI finally knows you" is the north star — the portrait must look good *and* work (Brief §3).

**Why (cites):** FR-SA5–8, NFR-SA2/SA3 (PRD `85qj2wwU2L`); FR-M1–4/FR-L1–5/FR-T1–4, AD-1/2/3/4/12 (Memory gap-closure fold-in, PRD `g9vWbSYplh`, spine `iiCcKhGptV`); A-PRIVACY (arch `ekTh7nDQqo`).
