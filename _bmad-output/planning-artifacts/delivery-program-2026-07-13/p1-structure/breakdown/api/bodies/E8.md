## 🗺️ Epic — Your Wiki & the DocmostPort

The wiki is the kept-knowledge layer — "your HDD." Studio consumes it as a programmatic client of the orvex-wiki engine. This epic owns the **DocmostPort** (FR-SA23) — the typed client for the engine's full surface Studio depends on: createSpace / createPage / updatePage-with-CAS (`LostUpdateError`) / archive / restore / purge-gated / getPageHistory / listPages / mintMagicOtp-transport / recentPagesForUser — used by Curator applies (as-user), Demo Engine calls, capture, and the read pane, with the engine-enforced **over-quota 402** (`QUOTA_EXCEEDED`) handled as a first-class outcome (surface the upgrade moment, stage the pending proposal, never retry-loop).
