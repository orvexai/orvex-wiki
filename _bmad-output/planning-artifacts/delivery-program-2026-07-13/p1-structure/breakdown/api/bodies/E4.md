## 🗺️ Epic — The Prompt Composer & enrichment

The Prompt Composer is the payoff surface — "prompts are the payoff, context is the product." This epic owns the **enrichment orchestration** at every Prompt-Use surface: retrieval-and-templating on the hot path, **LLM-free** by architecture (Phase-2 constraint, binding), sources = Memory (first-class, local) + wiki (via knowledge), relevance-ranked under a context budget, **≤500ms p95 added, over-budget ⇒ ship unenriched, never stall** (FR-SA9 / NFR-SA1). It owns the **Private-Memory guard** — private items never enrich silently; a per-use ask names the specific item(s), governed by the per-user Enrichment Policy (ask-each-time / never-use / always-allow), and every enriched egress logs which items were woven for the consent/audit trail (FR-SA10). And it owns **degradation-first** behaviour: knowledge down ⇒ Memory-only enrichment via the deterministic path; below relevance threshold ⇒ no injection, never noise; no user-facing errors in degraded modes (FR-SA11 / NFR-SA5).

The enriched artifact is transient: the stored prompt is always the clean, unenriched one (enrichment never persists). This is the £7 Composer that *supersedes* the Free FR-31 weave (E3-S5) only when active.

**Why (cites):** FR-SA9–11, NFR-SA1/SA5 (PRD `85qj2wwU2L`); Brief §4 "Consumption — the Prompt Composer … the most amazing prompt editor in the world."
