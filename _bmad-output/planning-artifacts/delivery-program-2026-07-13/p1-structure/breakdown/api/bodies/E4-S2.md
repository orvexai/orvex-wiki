## 🎯 Story

As a **privacy-conscious user**, I want a **Private-Memory guard with per-use consent and an egress audit trail**, so that private items never enrich silently — I am asked, naming the specific item(s), and every enriched egress is logged.

**Definition of Done:** one named test **`TestPrivateEnrichmentRequiresPerUseConsent`** (integration layer — runs enrichment where a relevant item is Private and asserts a per-use ask naming that item is required before injection, governed by the Enrichment Policy, and that the egress logs exactly which Memory/wiki items were woven, through the enrichment + consent interfaces). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2098); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria
- [ ] **AC1** — Given a relevant Private item, When enrichment would use it, Then a per-use ask naming the specific item(s) is required first. *Assertion: private item never injected without a per-use ask.* [Source: FR-SA10]
- [ ] **AC2** — Given the per-user Enrichment Policy, When set, Then ask-each-time (default) / never-use / always-allow govern the ask. *Assertion: policy value changes the ask behavior accordingly.* [Source: FR-SA10]
- [ ] **AC3** — Given an enriched egress, When it ships, Then it logs which Memory/wiki items were woven (consent/audit trail). *Assertion: audit records the exact woven item set.* [Source: FR-SA10, NFR-SA2]
- [ ] **AC4** — Given the audit trail, When written, Then it is never surfaced mid-flow (recorded, not shown inline). *Assertion: audit is out-of-band, not in the user-facing enrichment output.* [Source: FR-SA10]
- [ ] **AC5** (negative) — Given `never-use` policy, When a Private item is relevant, Then it is never injected and no ask is shown. *Assertion: never-use → zero private injection, zero ask.* [Source: FR-SA10]
- [ ] **AC6** (error-path) — Given a declined per-use ask, When enrichment proceeds, Then the item is excluded and the prompt ships without it. *Assertion: decline → item excluded, no stall.* [Source: FR-SA10]
