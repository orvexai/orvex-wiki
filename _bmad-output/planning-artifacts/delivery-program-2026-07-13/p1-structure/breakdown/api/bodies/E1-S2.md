## 🎯 Story

As a **Studio BFF**, I want **edge auth that verifies the caller against the shared security ceiling and resolves the polymorphic {user|org} principal**, so that every route runs under a correctly-keyed tenant and studio-api never grows its own auth authority.

**Definition of Done:** one named test **`TestEdgeAuthResolvesPolymorphicPrincipal`** (integration layer — drives a request with a valid delegated token through the auth middleware and asserts the resolved principal is `user:<id>` for a personal caller and `org:<id>` for a Teams caller, deny-by-default on missing/invalid scope, through the middleware interface). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2098); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria
- [ ] **AC1** — Given a valid token, When it hits the auth middleware, Then it is verified against the ceiling (multi-issuer JWKS + bounded cached introspection) via the ENG-2098-pinned TS verify mechanism. *Assertion: valid token → authenticated; expired/revoked → 401.* [Source: FR-SA29, A-AUTH]
- [ ] **AC2** — Given a personal caller, When the principal resolves, Then it is `user:<id>` with NO Clerk org; a Teams caller resolves to `org:<id>`. *Assertion: principal key matches `^(user|org):` and personal never carries an org.* [Source: FR-SA29, D-S17]
- [ ] **AC3** — Given deny-by-default scopes, When a request lacks the required scope, Then it is rejected before any handler runs. *Assertion: missing scope → 403, handler not invoked.* [Source: FR-SA29, A-AUTH]
- [ ] **AC4** — Given a destructive/admin surface, When accessed, Then `RequireFresh` is enforced. *Assertion: stale-auth request to a RequireFresh route → 401/step-up.* [Source: FR-SA29]
- [ ] **AC5** (negative) — Given a Team org principal, When a member's Memory is requested, Then the org principal does not grant member-Memory visibility. *Assertion: cross-member Memory read → denied (user is the privacy axis).* [Source: FR-SA29, NFR-SA3]
- [ ] **AC6** (error-path) — Given the interim Clerk-direct edge (OQ-SA9), When identity has not cut over, Then the exchange bridge path is the named interim and is CI-flagged as not-yet-cutover. *Assertion: interim path emits a deprecation/observability marker.* [Source: FR-SA29, OQ-SA9]
