## 🎯 Story

As a **Studio BFF**, I want to **publish typed CloudEvents to the Kafka-backed `studio-spine` through my own Postgres transactional outbox + relay**, so that domain state changes and their events commit atomically (no dual-write) and knowledge/billing get the canonical propagation feed from day one.

**Definition of Done:** one named test **`TestOutboxRelayPublishesAtomically`** (integration layer — writes a domain change + outbox row in one Postgres transaction, asserts the relay publishes exactly-once to the spine after broker ack, and that a rolled-back transaction publishes nothing, through the outbox/relay interface). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2098); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria
- [ ] **AC1** — Given a domain state change, When it is written, Then the change row and the outbox row commit in the SAME Postgres transaction. *Assertion: rolled-back change ⇒ zero outbox rows (no dual-write, CS §7).* [Source: FR-SA31, A-DATA §4]
- [ ] **AC2** — Given committed outbox rows, When the relay runs, Then it publishes to `studio-spine` after the broker acks and marks them sent. *Assertion: each event published exactly once; crash-replay re-publishes unsent only.* [Source: FR-SA31]
- [ ] **AC3** — Given a published event, When emitted, Then it is one of the typed `studio.*` CloudEvents (`user.provisioned/graduated`, `memory.updated` with privacy flag, `skill.published/updated`, `conversation.imported`, `card.staged/approved/rejected`) conforming to the ENG-2098-pinned schema. *Assertion: event validates against the contracts schema.* [Source: FR-SA31]
- [ ] **AC4** — Given every published event, When built, Then the shared envelope stamps the `orvexcell` extension attribute. *Assertion: no event lacks `orvexcell` (cell-contract rule 6).* [Source: A-CELL-COMPLIANCE]
- [ ] **AC5** (negative) — Given a direct-publish attempt (commit-then-publish), When code is scanned, Then it is absent — only the outbox path exists. *Assertion: no publish call outside the relay (grep + test).* [Source: A-DATA §4, CS §7]
- [ ] **AC6** (error-path) — Given a cell failover, When topics rebuild, Then they rebuild from the Postgres outbox (T1). *Assertion: replay from outbox reconstructs unsent events.* [Source: A-CELL-COMPLIANCE T1]
