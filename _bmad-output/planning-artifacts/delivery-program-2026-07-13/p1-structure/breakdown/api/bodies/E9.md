## 🗺️ Epic — Tiers, entitlements, MCP & workflow seams

The consumer/business/enterprise surfaces ride the **same microservices**; entitlements switch behaviour, not code (Brief §"three-surface arc"). This epic owns the **tier enforcement seam** (FR-SA27): consume billing's entitlement-check primitive (fast, cached, fail-consistent, principal-keyed on {user|org}) as the single tier authority — every £7 capability checks server-side at the API layer; the entitlement cache evicts on `billing.entitlement.changed` and studio-api re-pushes over one **SSE entitlement channel** (D-S20 — paywall dissolves <60s without reload); on billing outage, last-known within TTL, never fail-open to paid; dunning is 14-day grace → auto-downgrade to Free, never delete (D-S19). It owns the **product-edge allowances** (FR-SA28): demo ≈20, the 7-day card-required Personal trial, fair-use advisory, billing-state render, GBP-only pricing from billing's catalog (no price constants here), wiki-quota state render. It owns the **MCP-facing domain APIs** (FR-SA30): memory search/add/update (privacy-hard-excluded), capture/submit (through engineGate), capture_conversation — tier-enforced server-side (defense in depth) and rate-limited. And it owns the **step-APIs & callbacks for central workflows** (FR-SA32): provision/deprovision, Graduation, import, distillation, retention, purge — idempotent, dual-gated, with delegated tokens that never ride workflow inputs — plus the **degradation matrix** (NFR-SA5) and cost discipline (NFR-SA6).
