## 🎯 Story

As a **privacy-conscious user**, I want **3-state per-answer privacy (Open / Private / Shared-private) enforced through one consent pipeline and propagated to knowledge as a contract**, so that my private facts never leak into a prompt, over MCP, or into retrieval without a recorded per-use consent — SM-C3 = 0.

**Definition of Done:** one named test **`TestPrivateMemoryNeverLeaks`** (integration layer — sets a memory to Private and one to Shared-private, then adversarially probes default retrieval, MCP `search_memory`, and enrichment, asserting Private/Shared-private never appear without a recorded consent grant, and that a privacy-state change emits a re-index/eviction event before any query can serve the stale state, through the memory + event interfaces). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2098); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria
- [ ] **AC1** — Given a per-answer privacy state, When set, Then Open / Private (held back) / Shared-private (consent-granted) is authoritative over keyword auto-detection. *Assertion: user intent overrides auto-detected sensitivity.* [Source: FR-SA6]
- [ ] **AC2** — Given a sensitive answer, When processed, Then the `sensitive` → strip → held-back → consent pipeline (via `consent_events`, `sensitiveIncluded` allow-list) is the single enforcement mechanism. *Assertion: private data only escapes via a recorded consent event.* [Source: FR-SA6, NFR-SA2]
- [ ] **AC3** — Given consent withdrawal, When it occurs, Then the memory re-weaves + version-bumps and hard-delete is the prominent action. *Assertion: withdrawal → re-weave + version bump; hard-delete available.* [Source: FR-SA6]
- [ ] **AC4** — Given knowledge's Memory corpus, When indexed, Then per-user namespaces + privacy-state exclusion are first-class contract; Private/Shared-private never in default retrieval; MCP `search_memory` hard-excludes private. *Assertion: default retrieval + MCP search return zero private items.* [Source: FR-SA7, NFR-SA2/SA3]
- [ ] **AC5** — Given a privacy-state change, When applied, Then a re-index/eviction event is emitted BEFORE any subsequent query can serve the stale state. *Assertion: stale private state never served post-change.* [Source: FR-SA7]
- [ ] **AC6** (negative) — Given a same-org cross-member probe, When Memory is queried, Then no member sees another's Memory. *Assertion: cross-member Memory read → denied (user is the privacy axis).* [Source: NFR-SA3]
