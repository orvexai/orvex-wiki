As a **Studio user**, I want the **skill Item→Section→Skill model with a Base+Delta personalisation layer and an in-process deterministic render**, so that my skills render byte-stable in <50ms with zero model dependency — the Free-tier moat that works even with the AI down.

**Definition of Done:** one named test **`TestDeterministicRenderByteStable`** (unit layer — renders a fixed (FormSpec+values) skill twice and across the Base+Delta merge, asserts byte-identical output and <50ms, and asserts the `deterministic/` module cannot import the ai client, through the render function interface). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2098); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria
- [ ] **AC1** — Given the craft2 v3.2.3 model, When a skill is built, Then Item→Section→Skill structure, Base skill + Delta personalisation, and versioning persist. *Assertion: schema round-trips; Base immutable, Delta minted on fork/edit.* [Source: FR-SA1]
- [ ] **AC2** — Given identical (FormSpec+values), When rendered twice, Then output is byte-identical. *Assertion: `render(x) === render(x)` byte-for-byte.* [Source: FR-SA1, NFR-SA4]
- [ ] **AC3** — Given any skill, When rendered, Then it completes in <50ms with zero model egress. *Assertion: p95 render <50ms; `deterministic/` has no ai-client import (egress-throws gate).* [Source: FR-SA1, NFR-SA4, G2]
- [ ] **AC4** — Given a published-and-used skill, When deletion is attempted, Then it is refused. *Assertion: delete of published+used skill → error.* [Source: FR-SA1]
- [ ] **AC5** (negative) — Given the AI (orvex-studio-ai) is down, When a skill renders, Then it still renders identically. *Assertion: render succeeds with ai unreachable.* [Source: FR-SA1, G2]
- [ ] **AC6** (error-path) — Given a malformed FormSpec, When rendered, Then a typed validation error is returned (never a partial/garbage render). *Assertion: invalid input → typed error, no output bytes.* [Source: FR-SA1]
