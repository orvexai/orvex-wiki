## 🗺️ Feature area

The control-plane and compliance seams. This epic owns **edge auth** (`orvex-studio-lib/auth`, deny-by-default scopes) across billing's **three route classes** (bearer `/v1/*` tenant-scoped, admin `/admin/v1/*` console-fronted + `RequireFresh`, `WorkloadIdentityOnly` `/internal/v1/steps/*`, A-AUTH); the **workflow step-API surface** that orvex-studio-workflows drives all sagas through (idempotent, typed retryable/terminal envelope, D-WF-1 — billing originates no workflows); the **GDPR purge step** (cancel sub → delete Stripe Customer → pseudonymize journal + entitlement history with the Art. 17(3)(b) finance carve-out, A-GDPR); and the **entitlement-decision audit** append table + destructive-ops audit stream. Admin mutations (catalog edits, cap retunes, overrides, trial re-grants) are attributed entitlement versions like any other and fronted by orvex-studio-console's render-and-delegate pattern (console owns no store).

Governing canon: FR-B22/B23/B24/B25, A-AUTH, A-GDPR, D-WF-1, NFR-B3/B7.
