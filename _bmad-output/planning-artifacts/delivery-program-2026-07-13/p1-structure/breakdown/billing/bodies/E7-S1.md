## 🎯 Story

As **the billing service**, I want **edge auth via `lib/auth` across three route classes with tenant-scoped reads and console-fronted admin mutations**, so that **every surface is deny-by-default and admin overrides are attributed entitlement versions**.

**Definition of Done:** `TestThreeRouteClassesAuthEnforced` — a service-tier test asserting: `/v1/*` verifies via `lib/auth` with tenant-membership-scoped reads; `/admin/v1/*` requires admin scope + `RequireFresh` and is fronted by console; `/internal/v1/steps/*` is `WorkloadIdentityOnly`; and an admin override lands as an attributed entitlement version. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2100); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given `/v1/*` bearer surfaces, When called, Then they verify via `lib/auth` (identity JWKS + cached introspection, deny-by-default scopes) and reads are tenant-membership-scoped. *assert cross-tenant read → 403; missing scope → 401/403* [Source: FR-B24, A-AUTH]
- [ ] **AC2** — Given `/admin/v1/*` (catalog edits, cap retunes, overrides, trial re-grants), When called, Then admin scope + `RequireFresh` are required and the route is fronted by console's render-and-delegate wrap (console owns no store). *assert admin+RequireFresh enforced; non-admin → 403* [Source: FR-B24]
- [ ] **AC3** — Given an admin entitlement override, When applied, Then it is an entitlement version like any other: reasoned, attributed (admin actor), grandfather-safe. *assert override writes an attributed `source=admin_override` version* [Source: FR-B24, A-ENT]
- [ ] **AC4** — Given `/internal/v1/steps/*`, When called, Then it is `WorkloadIdentityOnly` + NetworkPolicy-restricted to workflows; `/admin/*` NetworkPolicy-open only to console's BFF. *assert workload-identity + NetworkPolicy enforced* [Source: FR-B24, A-AUTH]
- [ ] **AC5 (negative)** — Given an override rate above threshold, When it occurs, Then it alarms (grandfathering discipline decays socially; overrides are versioned, attributed, and alarmed). *assert override-rate alarm fires above threshold* [Source: FR-B24, arch §5]
