## 🎯 Story

As **one global Stripe account serving per-cell billing**, I want **a global-tier thin stateless forwarder that resolves each event's org → home cell and forwards the raw body untouched**, so that **the home cell's webhook binary is the sole verifier and no cross-cell reads hit the check path (A-CELL-ROUTING)**.

**Definition of Done:** `TestForwarderResolvesHomeCellForwardsRawUntouched` — an integration test asserting the forwarder at `billing.orvex.ai` holds NO webhook secret and performs NO verification, reads only `metadata.principal`/customer id to resolve customer → tenant → home cell via the global control-plane registry, and forwards the RAW body + `Stripe-Signature` header untouched to that cell's internal ingest route. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2100); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a delivery at `billing.orvex.ai`, When the forwarder receives it, Then it holds no Stripe webhook secret and verifies nothing — it only resolves customer → tenant → home cell from the global registry. *assert no verification at the forwarder; registry lookup performed* [Source: FR-B10, A-CELL-ROUTING]
- [ ] **AC2** — Given the resolved home cell, When forwarded, Then the RAW body + `Stripe-Signature` header are passed untouched to that cell's internal webhook-ingest route (workload-identity + NetworkPolicy). *assert body byte-identical + header preserved* [Source: FR-B10, A-CELL-ROUTING]
- [ ] **AC3** — Given the home cell, When it receives the forward, Then IT is the sole signature verifier and runs A-HOOK locally; a double-forward is a constant no-op (idempotent by Stripe event id). *assert home-cell verifies; double-forward ⇒ 200 no-op* [Source: FR-B10, A-CELL-ROUTING]
- [ ] **AC4** — Given single-cell launch, When routing runs, Then the registry lookup executes degenerate-locally (the seam is exercised before a second cell exists). *assert routing path invoked even single-cell* [Source: FR-B10]
- [ ] **AC5 (negative)** — Given a registry outage, When a new event arrives, Then new-event routing stalls (Stripe 72h retry + reconcile sweep absorb it) but in-cell entitlement serving is unaffected. *assert check path unaffected during registry outage* [Source: FR-B10, arch §5]
