## 🎯 Story

As **a user whose subscription changes**, I want **each lifecycle transition to flip my entitlement exactly once and serve authoritative billing-state**, so that **subscribe/upgrade/downgrade/cancel/payment-failure take effect without re-login (NFR-B2)**.

**Definition of Done:** `TestFlipSemanticsOneVersionPerTransition` — a service-tier test asserting each Stripe lifecycle transition maps to exactly one `entitlement_versions` write + FR-B7 propagation, is idempotent, effective without re-login (caches evicted), and `GET /v1/billing-state/{principal_type}/{principal_id}` serves the derived state (plan, renewal date, trial days, manage-link, dunning notice). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2100); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a lifecycle transition (subscribe/upgrade/downgrade/cancel/payment-failure), When applied, Then it maps to exactly one entitlement version write + one FR-B7 propagation. *assert one version + one propagation per transition* [Source: FR-B11, A-HOOK]
- [ ] **AC2** — Given a flip, When served, Then it is effective without re-login (server-side checks read the new version; caches evicted). *assert next check returns the new version, no re-auth* [Source: FR-B11]
- [ ] **AC3** — Given `customer.subscription.deleted`, When processed, Then the entitlement flips to Free, data intact (G5). *assert Free flip + no deletion* [Source: FR-B11, FR-B10]
- [ ] **AC4** — Given `invoice.payment_failed`, When processed, Then a dunning entry + `billing.payment.failed` audit event; the entitlement holds through a 14-day `past_due` grace then auto-downgrades to Free (never deleted). *assert 14-day grace then Free* [Source: FR-B11, FR-B10, D-S19]
- [ ] **AC5** — Given `GET /v1/billing-state/{principal_type}/{principal_id}`, When called, Then it returns plan, renewal date, trial days remaining, manage-link, and any dunning/SCA notice — server-derived, not client-computed. *assert billing-state shape served from billing* [Source: FR-B11]
- [ ] **AC6 (negative)** — Given a duplicate lifecycle event, When re-applied, Then no second version write (idempotent flip). *assert idempotent no-op on replay* [Source: FR-B11, NFR-B3]
