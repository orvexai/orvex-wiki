## 🎯 Story

As a **pricing surface (orvex-studio-ui) and every upgrade surface**, I want **billing to own the plan catalog as versioned data and serve it from one public read**, so that **every surface renders identical plans and enterprise is never leaked**.

**Definition of Done:** `TestPlanCatalogPublicReadExcludesEnterprise` — a service-layer test through the HTTP handler asserting `GET /v1/plans` returns `free`/`personal`/`teams` with their pinned feature sets + named caps, and that `enterprise` is absent from the response body (server-side exclusion, not client filter). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2100); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given the seeded catalog, When `GET /v1/plans` is called, Then the response lists `free`, `personal` (monthly £7 + annual £70), and `teams` (display-only, no price/purchase path), each with its plan-version feature set + named caps. *assert `body.plans[].code` set == {free, personal, teams} and each carries `plan_version` + `caps{}`* [Source: FR-B1]
- [ ] **AC2** — Given `enterprise` exists in the catalog, When `GET /v1/plans` is called on the public route, Then enterprise is excluded at the API layer. *assert no plan with `code == "enterprise"` in body* [Source: FR-B1]
- [ ] **AC3** — Given a plan-version pins a feature set + caps, When the catalog is read, Then each plan-version is immutable data referenced by id (a retune = new version, never an UPDATE). *assert repeated reads return the same `plan_version` id and a cap change produces a new version row* [Source: FR-B1, A-ENT]
- [ ] **AC4 (negative)** — Given a request with no valid bearer, When `GET /v1/plans` is called, Then it still succeeds (public catalog) but a malformed catalog row (missing caps) fails closed with 500 and an alert, never a partial plan. *assert 200 for anon; assert 500 + no partial plan on a caps-less row* [Source: FR-B1, CS §11]
