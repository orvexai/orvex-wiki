## 🎯 Feature area — Persistence, cell-contract conformance & security ops

The operational spine that makes identity a safe always-warm auth authority and the **reference cell-contract recipe every satellite copies** (identity is the cell-claim minter/registry owner — the miss propagates if not fixed here first). Persistence is **Postgres (RLS + JSONB append tables) + Redis — Mongo struck (D-S12)**; identity's own lifecycle-event journal is a range-partitioned Postgres append table with DROP-PARTITION retention. Health is split (SE-Arch F4): `/livez` always-200, `/readyz` round-trips Postgres/Redis/JWKS, `/healthz` a CLI-doctor alias that must NOT gate readiness. `CELL_ID` is required in prod (fail-boot), `CLUSTER_NAME` distinct, both echoed (F3, cell-contract rule 4). Cell-guard/421 middleware + `Idempotency-Key` on internal step APIs (F9, rules 3/11). OTel/Prometheus feed orvex-studio-console; every mint/revoke/provision is audited. The three service-space ADRs (A-TOKEN/A-VERIFY, A-SESSION) are authored (F6/§7).

Why it exists: PRD §4 F6 (FR-I16/I18) + NFR-I5/I6; Architecture §6/§7; SE-Arch F3/F4/F6/F9.
