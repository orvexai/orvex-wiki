## 🎯 Feature area — Token authority (mint / scope / lifecycle)

Identity is the family's token authority: it **brokers and scopes** MCP/agent tokens per the tenant's IdP (Clerk machine/M2M tokens; Keycloak client-credentials/token-exchange — A-TOKEN), with claims **FROZEN AT MINT** so an org-A token can never later carry org-B context after an org-switch. Tokens are mintable from **both** front ends (`POST /v1/tokens`) — each treated as untrusted input — with grantable scope derived **server-side from authoritative sources only** (never client-supplied), enforced deny-by-default inside `orvex-studio-lib/auth`. Full lifecycle (list / rotate / revoke) with revocation reflected in introspection within the TTL.

Why it exists: PRD §4 F2 + Goals G2; Architecture A-TOKEN. This is the C5 space-scope the whole platform relies on. **OQ-I2** (which Clerk primitive gives frozen per-space scoping + revocation) is security-load-bearing and must resolve before the A-TOKEN freeze.
