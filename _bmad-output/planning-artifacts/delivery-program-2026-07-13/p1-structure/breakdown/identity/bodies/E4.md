## 🎯 Feature area — Engine session seam & dual-IdP login

The FR-15 seam, corrected: what LEAVES the engine is the **direct Clerk verification**; what the engine KEEPS (irreducible) is the **opaque exchange-token consume → session mint**. Identity verifies the IdP → mints a short-lived exchange token → the engine consumes it and sets the cookie on its own origin (A-SESSION). Native engine email/password login is **removed fully — hosted AND standalone, no break-glass native login (D-S3)**; interactive login is Clerk SSO or, for BYO-IdP self-hosted tenants, the relocated **OIDC-RP + SCIM** path (FR-I13, first-class Keycloak — AGPL basis D-P9, severing every `@docmost/*` import). Admin recovery is an **identity-side offline-sealed break-glass, console-plane only**. Admin config surfaces relocate behind admin-org authz.

Why it exists: PRD §4 F4 + Goals G5; Architecture A-SESSION / A-BREAKGLASS. **Identity GA is a hard launch prerequisite** — native-login removal (WS-6) is gated on identity's Clerk-exchange + FR-15 mint being GA + login-availability soak-tested (WS-8 before WS-6); there is no fallback issuer.
