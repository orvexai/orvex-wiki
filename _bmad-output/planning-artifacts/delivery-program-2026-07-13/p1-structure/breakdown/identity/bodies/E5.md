## 🎯 Feature area — Lifecycle callee & event spine

Identity is a **stateless callee** of orvex-studio-workflows (D-WF-1: all Temporal orchestration is central — identity runs no worker). It serves the **studio-side** provision/deprovision mirror (`POST /api/clerk/provision|deprovision {scope,id,mode}`, idempotent) — a **different shape** than the engine's own controllers, which the workflow drives in parallel. Auth is **workload identity (mTLS/SPIFFE) scoped to workflows + NetworkPolicy** — never a shared static bearer. `deprovision(delete)` is **two-phase**: identity only soft-disables; the irreversible purge is gated behind the workflows grace timer + out-of-band confirmation. Identity emits its OWN lifecycle events onto the Kafka-backed studio-spine via a **transactional outbox + relay publishing DIRECT to Kafka** (D-S13 launch prerequisite; Redis→Kafka bridge RETIRED). The scaffold's CloudEvent-consuming `cmd/deprovisioner` (F2 double-ownership defect) is retired in favor of the REST step-API.

Why it exists: PRD §4 F5; Architecture A-LIFECYCLE; SE-Arch F2/F9.
