## 🎯 Feature area — Memory security & compliance (owned slice)

Identity's slice of the family Memory security fold-in (source: PRD `g9vWbSYplh` + Spine `iiCcKhGptV` + Map `vBvVDFklZo`). Identity owns **F6 security & compliance**: **BYOK** (customer-managed encryption keys via KMS, per-tenant, enforced at the store layer — for Memory + wiki content, at least for Teams/regulated users), a path to **SOC 2 Type II + HIPAA (BAA)**, a **self-host / BYOC** option for residency/air-gap needs, and the **personal↔employer firewall + per-use consent** enforced end-to-end. The keystone is **AD-8 — consent binds at the mint boundary, not the client compose**: a delivery fetch mints an **identity-owned, single-target, single-use, short-TTL delivery token** scoped `{memory, target, user}`; the untrusted client cannot compose without presenting one, and every per-use action re-mints (a ChatGPT token does not deliver to Gemini).

Why it exists: PRD "Memory gap-closure — owned slice" + Architecture AD-8/AD-6. Isolation-grade (FR-S4) resolves to the **stronger wall for regulated tenants**; identity supplies the polymorphic tenant key + BYOK store enforcement (the corpus isolation itself is knowledge's, the poisoning gate is staging's — identity's share is the key + consent boundary).
