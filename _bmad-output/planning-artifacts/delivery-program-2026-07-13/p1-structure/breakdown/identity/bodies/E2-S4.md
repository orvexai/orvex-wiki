## 🎯 Story

As a **user or admin**, I want to list, rotate, and revoke tokens, with revocation reflected in introspection within the TTL and emitting a lifecycle event, so that a compromised or unwanted token stops being honored promptly.

**Definition of Done:** `TestTokenLifecycle` (integration layer) — through `GET/DELETE /v1/tokens`, minted tokens are listed, rotation issues a new grant and invalidates the old, and revocation reflects in introspection within the TTL and emits a `token.revoked` lifecycle event.
*Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2101); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given minted tokens, When listing, Then all live grants for the caller are returned (no other tenant's). *assert: `GET /v1/tokens` returns caller's grants only* [Source: FR-I7]
- [ ] **AC2** — Given a token, When rotated, Then a new grant is issued and the old is invalidated. *assert: old token introspects inactive, new active* [Source: FR-I7]
- [ ] **AC3** — Given a token, When revoked, Then introspection returns inactive within the TTL. *assert: `introspect(revoked)` at ≤ TTL → active=false* [Source: FR-I7, NFR-I3]
- [ ] **AC4** — Given a revocation, When it commits, Then a `token.revoked` lifecycle event is emitted. *assert: outbox row / spine event `token.revoked` present* [Source: FR-I7, NFR-I3]
- [ ] **AC5** (negative) — Given a caller trying to revoke another tenant's token, When requested, Then it is refused with a uniform response (no existence oracle). *assert: cross-tenant revoke → constant 200/404-uniform, no leak* [Source: FR-I7, NFR-I4]
