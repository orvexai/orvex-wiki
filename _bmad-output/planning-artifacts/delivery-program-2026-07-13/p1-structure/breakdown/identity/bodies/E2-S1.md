## 🎯 Story

As the **token authority**, I want to mint MCP/agent tokens with claims **frozen at mint** per the tenant's IdP, so that an org-A token can never later carry org-B context after an org-switch and its scope claims are immutable post-mint.

**Definition of Done:** `TestTokenFrozenAtMint` (integration layer) — through the mint + verify path, `{tenant, clerk_org, subject, scopes, exp}` are fixed at mint and do not re-render from live session/active-org; a token minted under org A cannot verify against org B after an org-switch.
*Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2101); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a Clerk tenant, When a token is minted, Then it is a Clerk machine/M2M token with immutable custom claims. *assert: minted token claims == requested-at-mint, and are not JWT-template-rendered from session* [Source: FR-I4]
- [ ] **AC2** — Given a Keycloak tenant, When a token is minted, Then it is Keycloak client-credentials/token-exchange issued. *assert: `mint(kcTenant).idp == keycloak`* [Source: FR-I4]
- [ ] **AC3** (negative, the CI proof) — Given a token minted under org A, When the user switches to org B and the token is verified, Then verification against org B fails. *assert: `Verify(tokenA, ctxOrgB)` → `ErrOrgMismatch`* [Source: FR-I4, NFR-I4]
- [ ] **AC4** — Given a minted token, When claims are inspected after an org-switch, Then they are unchanged. *assert: claims immutable post-mint* [Source: FR-I4]
- [ ] **AC5** (error-path) — Given a cross-org mint attempt (mint for org B from an org-A session), When requested, Then it is rejected. *assert: cross-org mint → 403* [Source: NFR-I4]
