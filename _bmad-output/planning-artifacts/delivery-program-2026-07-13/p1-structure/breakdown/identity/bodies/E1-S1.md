## ✅ Acceptance Criteria

- [ ] **AC1** — Given a Clerk tenant, When a valid Clerk session JWT arrives, Then it verifies against Clerk's JWKS and returns the principal with `idp=clerk`. *assert: `resolve(clerkJWT).idp == "clerk" && .tenant == registry.tenantFor(clerkJWT.iss,sub)`* [Source: FR-I1]
- [ ] **AC2** — Given a Keycloak tenant, When a valid realm JWT arrives, Then it verifies against that realm's JWKS and returns `idp=keycloak`. *assert: `resolve(kcJWT).idp == "keycloak"`* [Source: FR-I1]
- [ ] **AC3** — Given a personal user-keyed tenant, When resolved, Then `org_or_realm` is empty/optional. *assert: `resolve(personalJWT).org_or_realm == ""`* [Source: FR-I1, D-S17]
- [ ] **AC4** (negative) — Given a token whose `iss` does not match the tenant's registered IdP, When verified, Then it is rejected. *assert: `resolve(mismatchIss)` returns error `ErrIssuerMismatch`, never a principal* [Source: FR-I1]
- [ ] **AC5** (error-path) — Given JWKS is unreachable and no cache entry exists, When verifying, Then fail closed (never fail-open to unauthenticated). *assert: `resolve()` returns error, principal is nil* [Source: FR-I1, FR-I3]
