## 🧭 Feature Area

`pkg/temporalutil` serves **both sides of the D-WF-1 split** without ever enabling an in-satellite worker (PRD F5 / FR-L18, FR-L19; Arch A-STEPAPI). The `temporalutil/worker` sub-package (client dial, worker bootstrap, activity helpers, obs interceptors) is imported **only** by orvex-studio-workflows — each satellite carries a CI grep forbidding the import. The `temporalutil/stepapi` sub-package (idempotency-key replay middleware; a typed retryable/terminal/success error envelope as the default response discipline; identity FR-I14's constant-200/no-existence-oracle shape as an explicit opt-in for destructive lifecycle endpoints only; the workload-identity caller hook shared with FR-L6's `WorkloadIdentityOnly`) is imported by satellites that serve step-APIs (knowledge rebuild, identity lifecycle).

Why it exists: D-WF-1 puts all Temporal central; satellites expose idempotent step-APIs instead of running workers. Central retries only work if activities classify retryable vs terminal vs success — or retries mask failures as false-green or never fire.
