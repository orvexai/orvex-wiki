## 🎯 Story

**As** wiki-api/mcp/ai, **I want** the engine's F-QUOTA HTTP 402 `QUOTA_EXCEEDED` rejection surfaced by `pkg/engineclient` as a typed, terminal, non-retryable error carrying the entitlement/upgrade deep-link verbatim, **so that** no consumer can accidentally retry, swallow, or soften a quota rejection and the upgrade path propagates unchanged (FR-L12a; Arch A-CLIENTS).

**Definition of Done:** ONE named test `TestQuotaExceededTerminalError` (unit, in `pkg/engineclient`, driving a write against a fake engine returning 402) asserting the client returns a typed non-retryable terminal error carrying the deep-link, that the retry machinery never retries it, and that it is never mapped to allow/partial. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2092); this story is dispatch-blocked until that tag exists.*
