## 🧭 Feature Area

The house telemetry and Kafka-first plumbing every satellite imports: `pkg/obs` gives one `Setup(ctx, serviceName)` that wires OTLP traces + metrics (+ logs) to the cluster otel-collector, W3C propagation, runtime metrics, graceful flush, HTTP server middleware + client transports so cross-satellite traces stitch, and a `/healthz`+`/livez` helper that echoes `CELL_ID` + `CLUSTER_NAME` (PRD F6 / FR-L20, FR-L21; Arch A-OBS, cell Rule 4). `pkg/kafkautil` gives standalone **assigned** consumers (`assign()` + `seek`, never a shared group so every replica sees all events), `<topic-epoch>:<offset>` cursor helpers with a single typed `ErrGap`, and `{domain}-events.{cell}` single-partition, cell-opaque topic naming (PRD F7 / FR-L22; Arch A-KAFKA, cell Rule 5). Kafka-first (USER 2026-07-05): both are launch-line — franz-go lands with knowledge's SSE gateway, the first real consumer.

Why it exists: orvex-studio-console fronts Loki/Mimir/Tempo — `pkg/obs` is the producer side of that contract. lib carries **no** engine outbox-relay plumbing (canon P2: outbox→relay→Kafka; the relay is infra, not lib code).
