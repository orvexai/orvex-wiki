## 🎯 Story

**As** a satellite build agent, **I want** every lib package to ship fakes/test doubles (fake introspection server, static-JWKS signer for authtest, in-memory engine/knowledge/billing/identity clients, in-memory Kafka records, fake Broker ingress, fake OTLP collector), **so that** satellites can unit-test their seams offline without any live infrastructure and the family's isolated-build doctrine holds (NFR-L5; Arch §5).

**Definition of Done:** ONE named test `TestPackageFakesOffline` (unit, exercising each shipped fake through the package's public interface) asserting a consumer can wire auth middleware, obs, and the typed clients against the fakes with no live infra, and that each fake faithfully reproduces the real contract's success + failure paths. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2092); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given each package, When a satellite imports its fake, Then it can unit-test the seam offline (no live infra). *assert every package exports a usable double* [Source: NFR-L5]
- [ ] **AC2** — Given the auth fakes (static-JWKS signer + fake introspect/registry), When authtest runs, Then it verifies against them with no IdP/identity network hop. *assert offline conformance run* [Source: NFR-L5 / FR-L7]
- [ ] **AC3** — Given the client fakes (in-memory engine/knowledge/billing/identity), When exercised, Then they reproduce success AND failure/fail-closed paths. *assert both paths present per fake* [Source: NFR-L5]
- [ ] **AC4** — Given the events/kafka fakes (fake Broker ingress, in-memory Kafka records) and obs fake (fake OTLP collector), When exercised, Then publish/consume/telemetry are testable offline. *assert each testable offline* [Source: NFR-L5]
- [ ] **AC5** — Given a new package method, When added, Then its fake is updated in the same change (fakes track the surface). *assert fake-parity discipline* [Source: NFR-L5]
