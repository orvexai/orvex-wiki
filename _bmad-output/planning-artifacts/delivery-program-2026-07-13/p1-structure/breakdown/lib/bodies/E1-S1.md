## 🎯 Story

**Audit: already implemented** (evidence in §4) — this story is VERIFY + harden: prove the behaviour via the named DoD test + adversarial review; do not rebuild.

**As** every Go satellite (and identity itself), **I want** one dual-IdP, multi-issuer verifier that turns a raw session/MCP/api token into a canonical IdP-agnostic principal, **so that** satellite code never branches on IdP, never sees raw claims, and the whole family runs byte-identical verification (FR-L1; Arch A-VERIFY-IMPL steps 1–3).

**Definition of Done:** ONE named test `TestVerifierCanonicalPrincipal` (unit, in `pkg/auth`, driven through the exported `Verify(ctx, rawToken)` interface against a static-JWKS signer and the fake issuer registry) asserting a valid Clerk and a valid Keycloak token each yield `{idp, subject, tenant, org_or_realm, roles, token_scope, exp}` with `cell`/`cell_epoch` parsed when present, and that an `iss` not matching the tenant's registered IdP is rejected. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2092); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a valid token signed by the tenant's registered issuer, When `Verify` runs, Then it returns the canonical principal and no raw IdP claim escapes to handler context. *assert `principal` has exactly the 7 canonical fields (+optional cell pair) and `ctx` carries no `raw_claims` key* [Source: FR-L1]
- [ ] **AC2** — Given a token whose `iss` is not in identity's registry OR not the tenant's registered IdP, When `Verify` runs, Then it rejects. *assert `err == ErrIssuerNotRegistered`, principal nil* [Source: FR-L1 / Arch V2]
- [ ] **AC3** — Given a short-TTL session/access token carrying `cell`/`cell_epoch`, When `Verify` runs, Then both are parsed onto the principal; Given a long-lived bearer/MCP token with neither, Then their absence is **not** a verification failure. *assert principal.cell set in case A; err nil and principal.cell zero in case B* [Source: FR-L1 / cell contract Rule 2, Token rule]
- [ ] **AC4** (negative) — Given an invalid signature, expired, or malformed token, When `Verify` runs, Then it rejects fail-closed. *assert reject for each; never returns a principal on error* [Source: FR-L1 / Arch V1]
- [ ] **AC5** (perf) — Given a warm per-issuer JWKS cache, When `Verify` runs, Then it makes **no** identity/IdP network hop and completes < 5 ms. *assert benchmark p99 < 5 ms with zero outbound calls recorded on the fake* [Source: NFR-L1]
