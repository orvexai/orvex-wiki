## 🧭 Feature Area

`pkg/dfm` is the family's **only** DfM↔ProseMirror serializer outside the AGPL engine — a clean-room Go implementation shared by orvex-wiki-api and the Orvex CLI (PRD F8 / FR-L26, FR-L27, FR-L28; Arch A-DFM). It exports exactly four functions (`PMToDfM`, `DfMToPM`, `MarkdownToPM`, `MarkdownToDfM`); conversion is in-process for both embedders (neither calls any engine `/v1/convert`). Block types DfM cannot render round-trip through **opaque-node handles** so an edit *around* a block never corrupts it; the package invariant `DfMToPM(PMToDfM(doc)) ≡ doc` (up to declared normalizations) is pinned by tests. Provenance is load-bearing: written only from the documented ProseMirror schema + the engine's published block-schema catalog (`GET /api/schemas/blocks`), **never** from `@docmost/editor-ext` (AGPL §13). The legacy docmost-cli hand-mirrored serializer it replaces must pass an **AGPL-provenance audit before this package lands** (WS-11 pre-task, FR-L27), and a CI parity gate against the engine catalog + contracts golden fixtures keeps the two independent implementations from drifting (FR-L28).

Why it exists: the engine's serializer is AGPL and cannot be imported into a closed program; the CLI's ported serializer is exactly the contamination path the AGPL boundary forbids (map cross-cutting risk).
