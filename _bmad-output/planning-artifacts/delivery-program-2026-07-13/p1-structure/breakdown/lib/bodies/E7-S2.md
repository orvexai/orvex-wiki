## 🎯 Story

**As** a consuming satellite, **I want** lib released as semver git tags on the single module `github.com/orvexai/orvex-studio-lib` (a `v0.x` pre-stabilization window), where the public API keeps compiling within a minor, every release carries a changelog, and an apidiff compile-compat check runs in CI, **so that** a satellite pins a tag and states exactly which contract snapshot + lib version it speaks, and an API break is a deliberate changelogged event (FR-L23, NFR-L3; Arch A-VERSION).

**Definition of Done:** ONE named test/gate `TestAPIDiffCompileCompat` (CI gate, run in lib CI) asserting that the public API compiles against the previous minor (apidiff), that a breaking change is flagged (major, or `v0.x` minor with a flag), and that the lib tag and the contracts pin are independently visible in the module. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2092); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a new release, When cut, Then it is a semver git tag on the single module with a changelog entry. *assert tag + changelog present* [Source: FR-L23]
- [ ] **AC2** — Given the public API, When apidiff runs vs the previous minor, Then within a minor it keeps compiling for consumers. *assert compile-compat within minor* [Source: FR-L23 / NFR-L3]
- [ ] **AC3** (negative) — Given a breaking API change, When CI runs, Then it is flagged as major (or `v0.x` minor with a flag), never a silent break. *assert break ⇒ flagged* [Source: FR-L23 / NFR-L3]
- [ ] **AC4** — Given a deployed satellite, When inspected, Then the lib tag and the contracts pin it carries are independently visible. *assert both versions surfaced* [Source: FR-L23 / NFR-L3]
- [ ] **AC5** — Given an emergency `pkg/auth` (incl. cell-guard) security fix, When released, Then it ships as a patch on the current minor and is announced family-wide. *assert patch-release path exists* [Source: FR-L23 / Arch §4]
