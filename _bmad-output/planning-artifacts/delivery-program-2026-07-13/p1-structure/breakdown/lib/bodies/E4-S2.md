## 🎯 Story

**As** a satellite serving a step-API (knowledge rebuild, identity lifecycle), **I want** `pkg/temporalutil/stepapi` server helpers in three separable pieces — idempotency-key replay middleware for every step-API; a typed retryable/terminal/success error envelope as the **default** response discipline; and the constant-200/no-existence-oracle shape as an explicit opt-in for destructive lifecycle endpoints only — plus the workload-identity caller-authentication hook, **so that** central Temporal can classify failures for retries and destructive paths hide record existence without making that the package-wide default (FR-L19; Arch A-STEPAPI).

**Definition of Done:** ONE named test `TestStepAPIIdempotencyAndErrorEnvelope` (unit, in `pkg/temporalutil/stepapi`, driving handlers through the middleware) asserting a replayed idempotency-key does not re-execute (record retention ≥ max Temporal retry window), that the default envelope distinguishes retryable/terminal/success, and that constant-200 applies only to opt-in destructive endpoints. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2092); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a step-API call with a repeated `Idempotency-Key`, When replayed, Then the handler does not re-execute and returns the recorded result. *assert single execution; retention ≥ max retry window* [Source: FR-L19]
- [ ] **AC2** — Given the default response discipline, When a handler returns, Then the typed envelope distinguishes retryable failure / terminal failure / success-with-payload. *assert three-way classification present* [Source: FR-L19]
- [ ] **AC3** — Given a destructive lifecycle-mirror endpoint that opts in, When it returns, Then it uses the constant-200/no-existence-oracle shape; Given a non-opt-in endpoint, Then it uses the default envelope. *assert opt-in scoping, not package-wide* [Source: FR-L19 / identity FR-I14]
- [ ] **AC4** (auth) — Given a destructive path, When called, Then the workload-identity hook authenticates the caller (mechanism per OQ-L6), backing FR-L6's `WorkloadIdentityOnly` class. *assert caller check enforced* [Source: FR-L19 / FR-L6]
- [ ] **AC5** (negative) — Given an expired idempotency key, When replayed, Then it never causes silent re-execution. *assert no re-execution past retention boundary handling* [Source: FR-L19]
