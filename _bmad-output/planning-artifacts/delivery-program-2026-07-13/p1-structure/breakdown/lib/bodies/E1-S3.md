## 🎯 Story

**Audit: already implemented** (evidence in §4) — this story is VERIFY + harden: prove the behaviour via the named DoD test + adversarial review; do not rebuild.

**As** the verifier, **I want** a push-revocation deny-list that consumes identity's revocation broadcast into a Redis-backed, tenant-keyed, exp-driven list consulted on every verification, **so that** revocation does not depend on each satellite reaching introspection and a revoked long-lived MCP/api token stays denied for its full remaining lifetime (FR-L4; Arch A-DENYLIST).

**Definition of Done:** ONE named test `TestDenyListRevocationCeiling` (unit, in `pkg/auth`, driving the mounted `auth.RevocationHandler` then `Verify`) asserting that a broadcast revocation writes `deny:{tenant}:{jti}` with TTL = the token's remaining `exp`, that every subsequent verification is rejected, and that the per-replica in-memory front can serve "not denied" for at most the ≤ 2 s negative-cache window. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2092); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a revocation broadcast delivered to the `RevocationHandler`, When it processes, Then it writes `deny:{tenant}:{jti}` with TTL = remaining `exp`. *assert Redis key + TTL within 1 s of `exp-now`* [Source: FR-L4 / NFR-L6]
- [ ] **AC2** — Given a revoked token, When `Verify` runs on any replica, Then it is rejected (deny-list consulted every request, O(1)). *assert reject; assert 1 deny-list lookup per verify* [Source: FR-L4]
- [ ] **AC3** — Given a replica whose in-memory front has not been push-invalidated (Trigger hit a sibling replica), When it serves, Then it can hold "not denied" for at most ≤ 2 s before re-consulting Redis. *assert front TTL constant ≤ 2 s, not configurable up* [Source: FR-L4 / Arch A-DENYLIST]
- [ ] **AC4** (negative) — Given the revocation payload, When it omits the tenant, Then the handler rejects it (broadcast must carry the tenant, contracts FR-C6 obligation). *assert handler error on tenant-less payload* [Source: FR-L4 / NFR-L6]
- [ ] **AC5** — Given a cold-starting replica, When it verifies, Then it sees the Redis deny-list immediately (no per-replica replay gap). *assert reject on first verify after cold start* [Source: FR-L4 / Arch A-DENYLIST]
