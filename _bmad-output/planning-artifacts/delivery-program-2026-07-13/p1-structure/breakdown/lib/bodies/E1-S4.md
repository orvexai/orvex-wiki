## 🎯 Story

**Audit: already implemented** (evidence in §4) — this story is VERIFY + harden: prove the behaviour via the named DoD test + adversarial review; do not rebuild.

**As** a satellite exposing high-privilege operations (token mint, provision/deprovision, admin, any cross-tenant action), **I want** a `RequireFresh` API that performs non-cached, no-grace introspection, **so that** the most dangerous operations never serve from a cache or grace window and identity-down means the operation fails full-stop (FR-L5; Arch A-VERIFY-IMPL step 7, §3 V7).

**Definition of Done:** ONE named test `TestRequireFreshNoGrace` (unit, in `pkg/auth`, driving a `RequireFresh`-declared route through the middleware against a fake introspect server) asserting the route always re-introspects (never cache-hits) and rejects when identity is unreachable, and that a route with a scope declaration but **no** privilege level is rejected at startup/CI. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2092); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a `RequireFresh` route, When a request arrives, Then a fresh introspection call is made even with a warm cache. *assert 1 introspect call per request, cache bypassed* [Source: FR-L5]
- [ ] **AC2** (error path) — Given identity unreachable on a `RequireFresh` route, When a request arrives, Then it fails closed — no grace, no cache fallback. *assert reject, `err == ErrIntrospectUnavailable`* [Source: FR-L5 / Arch V7]
- [ ] **AC3** (negative) — Given a route with a scope declaration but no privilege level, When the router mounts, Then it is rejected at startup/CI exactly like an undeclared scope. *assert mount panics/returns config error; never silently degrades to per-request fresh at runtime* [Source: FR-L5]
- [ ] **AC4** — Given the privilege level is a required field of the per-route declaration, When a high-privilege op is declared, Then `RequireFresh` is enforced structurally. *assert declaration schema requires privilege level* [Source: FR-L5]
