## 🧭 Feature Area

The library-engineering spine that makes lib a disciplined, versioned, drift-proof artifact rather than a dumping ground (PRD F9 / FR-L23, FR-L24, FR-L25; NFR-L3/L4/L5/L7; Arch A-CODEGEN, A-VERSION, A-DEPS). It carries: the codegen relationship (generated contract types committed under `gen/`-suffixed trees, pinned to a contracts release tag in one in-repo manifest, regen-diff CI gate so hand-editing generated files cannot land — FR-L24); versioning (single Go module `github.com/orvexai/orvex-studio-lib`, semver git tags, `v0.x` window, apidiff compile-compat vs previous minor — FR-L23); the library shape (no `cmd/`, no Dockerfile, no deploy; heavy SDKs confined package-by-package so a `pkg/auth`-only importer links none of them — FR-L25/NFR-L4); the family AGPL-import CI guard (NFR-L7); and the offline test doubles every package ships (NFR-L5).

Why it exists: G3 (contract fidelity, CI-enforced) and G4 (cheap adoption) both depend on this spine; the counter-metrics (no business logic, bounded churn, no heavy-SDK bleed) are enforced here.
