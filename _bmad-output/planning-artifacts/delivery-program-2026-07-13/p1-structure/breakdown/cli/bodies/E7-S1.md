## 🎯 Story

**Audit: already implemented** (evidence in §4) — this story is VERIFY + harden: prove the behaviour via the named DoD test + adversarial review; do not rebuild.

As an **operator**, I want `orvex auth login/logout/whoami/status/use/list-profiles` routed to identity — minting an identity-scoped token and reading the caller principal/role from token claims — so that I authenticate once per service and `whoami` reflects my scoped identity without an engine `POST /api/users/me` probe.

**Definition of Done:** ONE named test `TestAuthLoginMintsScopedTokenAndWhoamiFromClaims` — an auth-layer test asserting, through the auth commands against a stub identity service, that `login` mints and stores a space-scoped token per service, `whoami`/`status` read principal/role from token claims (not an engine probe), and `use`/`list-profiles` switch/enumerate profiles. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2105); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given `auth login`, When run against identity, Then a space-scoped token is minted and stored per service on the Profile. *Assertion: token minted from identity; stored per-service (or one bearer for all edges).* [Source: FR-CLI17/FR-CLI5]
- [ ] **AC2** — Given `whoami`/`status`, When run, Then the caller principal/role are read from token claims — not an engine `POST /api/users/me`. *Assertion: identity read from claims; no engine probe.* [Source: FR-CLI17]
- [ ] **AC3** — Given `use`/`list-profiles`, When run, Then profiles switch/enumerate. *Assertion: active profile changes; list returns configured profiles.* [Source: FR-CLI5]
- [ ] **AC4 (error-path)** — Given identity unreachable, When `login` runs, Then a typed `IDENTITY_UNREACHABLE`/`AUTH_*` error, never a silent partial login. *Assertion: typed error; no half-stored credential.* [Source: NFR-CLI4]
