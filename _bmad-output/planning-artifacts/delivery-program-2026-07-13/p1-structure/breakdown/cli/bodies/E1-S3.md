## 🎯 Story

As a **CLI developer**, I want one service-aware transport that every callsite funnels through — resolving base URL per service, applying per-service envelope decode, keeping retry/backoff/`Retry-After`, originating a correlation ID, and minting an `Idempotency-Key` on replayable writes — so that base-URL, envelope, error, and observability behaviour is a one-place change instead of ~90 drifting callsites.

**Definition of Done:** ONE named test `TestTransportMintsIdempotencyKeyAndPropagatesCorrelationID` — a transport-layer test asserting, through the transport's public `Do(service, req)` interface against a stub server, that a replayable write carries a stable `Idempotency-Key` across a retried transient 5xx and that a correlation ID is sent as a header and echoed in the error envelope as `request_id`. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2105); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a wiki-api call, When decoding, Then the transport applies the `{data}` envelope on `/api/orvex/*`, bare receipts `{url,id,version,persisted}` on `/v1`, and typed shapes on knowledge/ai. *Assertion: each service's response decodes via its declared envelope.* [Source: FR-CLI11; Arch Components: transport]
- [ ] **AC2** — Given a replayable write (create/upsert) that hits a transient 5xx, When retried, Then the same `Idempotency-Key` is sent so the write cannot double-apply. *Assertion: both attempts carry an identical `Idempotency-Key` header (cell contract §5 rule 11).* [Source: FR-CLI11; finding F-F]
- [ ] **AC3** — Given any outbound call, When issued, Then a correlation/request ID is originated at CLI ingress and propagated as a header, and echoed in the error envelope as `request_id`. *Assertion: header present on outbound; `request_id` present in error envelope (CS §10, finding F-G).* [Source: FR-CLI11]
- [ ] **AC4 (negative)** — Given a non-idempotent method on a 5xx, When the transport considers retry, Then it does NOT retry (retries confined to idempotent methods). *Assertion: no second attempt for a non-idempotent verb without an Idempotency-Key.* [Source: FR-CLI11; finding F-F]
- [ ] **AC5 (error-path)** — Given a 401/403/404 upstream, When the transport returns, Then it propagates verbatim with no existence oracle. *Assertion: status + errorCode passed through unchanged.* [Source: NFR-CLI4]
