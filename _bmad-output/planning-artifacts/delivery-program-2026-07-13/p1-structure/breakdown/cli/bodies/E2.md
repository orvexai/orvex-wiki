## 🎯 Epic — Output Contract, Error Vocabulary & Command-Surface Parity

The reference's chassis — JSON-when-piped / human-on-TTY output, `--output json|yaml|human`, snake_case fields, `--fields/--id-only/--compact` token-minimizers, `SetEscapeHTML(false)`, the **frozen 0–9 exit codes** and the **errorCode string vocabulary** — is the contract thousands of existing agent/script callers already branch on. This epic re-authors that chassis as a **shared client artifact** every namespace reuses, structures the cobra tree **service-first** (`wiki`/`search`/`ai`/`auth`/`admin` as service namespaces; `daemon`/`cache`/`config`/`doctor`/`audit`/`instructions`/`completion`/`version` as client-local groups) with one-release-window hidden aliases, ships the **two build variants** (`orvex` pure-Go + `orvex-full` chromium/treesitter) with honest `REQUIRES_FULL_BINARY` stubs, and stands up the **docmost-cli behaviour-parity harness as a CI gate** (G2). Callers must keep working byte-identically across the repoint — minus Linear. [Source: PRD R4AOVBLST7 §Goals G1/G2/F2/F3; Architecture pf10XC2Qjz §Tiers/§Components]

Why it exists: parity is the acceptance mechanism, not a spot-check (NFR-CLI1). Without a frozen, shared output/error layer and an automated oracle diff against `docmost-cli`, every repoint risks a silent contract regression that breaks downstream automation.
