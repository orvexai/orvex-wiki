## 🎯 Story

**Audit: already implemented** (evidence in §4) — this story is VERIFY + harden: prove the behaviour via the named DoD test + adversarial review; do not rebuild.

As a **CLI operator on a split/dev deployment**, I want an endpoint registry on the Profile that resolves a per-service base URL from an `ORVEX_URL` gateway shorthand or explicit per-service overrides, so that one binary reaches wiki-api, knowledge, events, ai, and identity on their correct hosts without a single baked `InstanceURL`.

**Definition of Done:** ONE named test `TestEndpointRegistryResolvesPerServiceHosts` — a unit-layer test asserting, through the registry's public `Resolve(service)` interface, that (a) `ORVEX_URL` derives each service host by the url-scheme convention and (b) an explicit `ORVEX_WIKI_API_URL` override wins. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2105); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given `ORVEX_URL` set and no overrides, When resolving each service, Then per-service hosts derive by the family url-scheme convention (D-S27: cell services `{service}.{cell}.orvex.{tld}`; tenant `{tenant}.wiki.{cell}.orvex.{tld}`). *Assertion: `Resolve("wiki")`, `Resolve("knowledge")`, `Resolve("events")`, `Resolve("ai")`, `Resolve("identity")` each return the convention-derived URL.* [Source: FR-CLI3]

- [ ] **AC3** — Given the host FORM is unpinned in canon (OQ-CLI2/2b: ai host, public knowledge query host, wiki-api flat vs `api.wiki`), When those hosts are requested without an override, Then the CLI surfaces a typed configuration error rather than guessing. *Assertion: `Resolve("ai")` with no pin returns `*_UNREACHABLE`/config error, no fabricated host.* [Source: FR-CLI3; OQ-CLI2 — TBD, pinned by the Definition Pack (ENG-2105)]
- [ ] **AC4 (error-path)** — Given neither `ORVEX_URL` nor an override for a required service, When resolving, Then a typed `INVALID_ARGS`/config error is returned, never an empty/`InstanceURL`-style host. *Assertion: error is typed, host string is not silently blank.* [Source: FR-CLI3]
