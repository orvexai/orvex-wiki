## 🎯 Story

**Audit: already implemented** (evidence in §4) — this story is VERIFY + harden: prove the behaviour via the named DoD test + adversarial review; do not rebuild.

As a **CLI user wanting fast reads**, I want a per-host background daemon owning one SQLite cache (WAL, single-writer goroutine, flock lifecycle) served over Unix-socket length-prefixed-JSON IPC, so that reads resolve from a warm local cache while the daemon stays non-load-bearing for correctness.

**Definition of Done:** ONE named test `TestDaemonServesReadsOverIpcSingleWriter` — a daemon-layer test asserting, through the Unix-socket IPC interface, that a read resolves from the SQLite cache, that the WAL cache enforces a single writer, and that a cold/stale cache yields a typed staleness error (never wrong or over-exposing data). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2105); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given the daemon running, When a read arrives over Unix-socket IPC, Then it resolves from the SQLite cache when warm. *Assertion: warm read served from cache, not upstream.* [Source: FR-CLI14]
- [ ] **AC2** — Given concurrent writes to the cache, When applied, Then a single-writer goroutine serializes them (WAL). *Assertion: no concurrent-writer corruption; single-writer invariant holds.* [Source: FR-CLI14]
- [ ] **AC3** — Given `--no-daemon` or a cold cache, When reading, Then the transport calls upstream directly (daemon non-load-bearing). *Assertion: cold path reaches upstream; correctness independent of daemon.* [Source: FR-CLI14]
- [ ] **AC4 (error-path)** — Given a stale/cold cache, When served, Then a typed staleness error is returned, never wrong or over-exposing data. *Assertion: typed staleness error; no stale-but-green.* [Source: FR-CLI14; NFR-CLI4]
- [ ] **AC5 (platform)** — Given install, When performed, Then daemon install is Linux/systemd-user only. *Assertion: systemd-user unit; no launchd/Windows.* [Source: FR-CLI14; Constraints]
