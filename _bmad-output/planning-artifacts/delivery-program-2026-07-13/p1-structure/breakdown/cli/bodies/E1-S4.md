## 🎯 Story

As a **CLI developer**, I want per-service typed clients generated only from the orvex-studio-contracts tag (never a served descriptor) with a CI drift gate, so that nothing AGPL-originated enters the codegen path and the CLI can only reflect what contracts author.

**Definition of Done:** ONE named test `TestGeneratedClientsMatchContractsTag` — a CI/unit-layer test asserting, through the generated client packages' public types, that the checked-in `gen/` trees are byte-identical to a fresh codegen run against the pinned contracts tag (drift → fail). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2105); this story is dispatch-blocked until that tag exists.*
