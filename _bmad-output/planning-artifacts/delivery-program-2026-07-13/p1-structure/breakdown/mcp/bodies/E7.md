## 🗺️ Feature area

This epic holds the non-functional spine that keeps "thin" honest: the **golden-tape KPI harness** (every common task ≤ 2 calls / ≤ ~1k tokens, live-counted) as a blocking CI + post-deploy release gate; **operability** (`/health` liveness-only, `/healthz` cell echo, readiness that degrades when the revocation consumer stalls, per-tool + shim-overhead + revocation-lag SLIs to the LGTM stack consumed by orvex-studio-console); the **AGPL-import guard** (this is the family's one TS repo where `import '@docmost/…'` is a one-liner, so the FR-C21 guard is load-bearing, not ceremonial) with typed-degradation posture (never fail open); and the long pole — **decommissioning the embedded `packages/orvex-mcp`** (73 tools) via the contracts FR-C5 REST-gap ledger, landing `list_tools` long-tail parity, and splitting the 2827-line `tools.ts` along the ADR-0004 tier boundaries so the "thin" claim is enforceable rather than asserted.

It exists because the KPI is the counter-metric to every "just add a helper tool" impulse (NFR-M6), and because a tool with no wiki-api/contracts/ai/knowledge operation behind it does not exist post-split (Arch §5 decommission parity).
