## 🗺️ Feature area

The wiki verb shim is the MCP's largest tool cluster: the read ladder (`get_page` info→outline→blocks), the write chokepoint (`save_page`, `edit`), pull-based history (`get_changes`), and navigation (`get_space_tree`, `get_neighborhood`). Each tool is protocol translation only — validate args → one/rarely-two typed-client calls → shape for the token budget → return typed errors verbatim (Arch A-SHIM). The truth-changing logic (block-patch orchestration, DfM serialization, ACL, atomicity) lives upstream in wiki-api; the MCP composes none of it (PRD §3 non-goals).

This epic exists to hold the context-light read/write ergonomics that make agents effective without inflating calls or tokens: the graduated read ladder that never dumps a whole page, the CAS-guarded write that returns a verified receipt (never a re-rendered body), and the verbatim passthrough of `409 VERSION_MISMATCH` / `402 QUOTA_EXCEEDED`. It is the surface the golden-tape KPI (E7-S1) measures hardest.
