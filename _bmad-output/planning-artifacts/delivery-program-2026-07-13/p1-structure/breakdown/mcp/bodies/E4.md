## 🗺️ Feature area

The auth edge is the MCP's trust boundary: it verifies every caller token per the **identity / lib-auth verifier contract** (multi-issuer JWKS for Clerk global + Keycloak realm, deny-list, cached introspection, deny-by-default scope gate) and emits the IdP-agnostic **org-scoped principal**. Because `orvex-studio-lib` is Go-only, the MCP carries the family's **one TypeScript verifier binding** — held to the Go semantics by the contracts FR-C13 conformance vectors as a blocking CI gate (Arch A-AUTH; the single place verifier logic exists twice in the family). It then forwards the caller's scoped token **verbatim** on every upstream, holds no service credential, transports human-minted RATIFY/CONFIRM tokens without ever minting or promoting, and carries the same posture on stdio.

It exists because the family invariant is "a token can never do through the MCP what it couldn't do directly" (PRD G3). Fail-open here is an incident by definition (counter-metric: fail-open = 0). This epic is the highest-risk correctness surface in the service.
