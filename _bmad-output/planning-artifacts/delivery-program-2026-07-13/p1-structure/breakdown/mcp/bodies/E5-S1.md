**Audit: already implemented** (evidence in §4) — this story is VERIFY + harden: prove the behaviour via the named DoD test + adversarial review; do not rebuild.

**As the** platform, **I want** the MCP to be a stateless shim — no Postgres/Mongo, kill-any-pod safe, its only caches principal-independent artifacts (authoring guide, tool metadata) plus the bounded auth caches the verifier contract mandates (JWKS, introspection TTL, revocation deny-list) — with the `jti` replay window retiring at R1 alongside the one-time session-resolve token class it guarded, **so that** correctness never depends on a cache and long-lived frozen-claim tokens are never wrongly replay-guarded.
