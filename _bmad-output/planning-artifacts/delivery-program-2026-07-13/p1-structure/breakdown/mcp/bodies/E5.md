## 🗺️ Feature area

The MCP is a stateless, kill-any-pod-safe shim: no Postgres/Mongo, no Temporal worker, no SSE fan-out. Its only caches are principal-independent artifacts (authoring guide, tool metadata) and the bounded auth caches the verifier contract mandates (JWKS, introspection TTL, revocation deny-list). Its **one resident subscription** is the cell-scoped revocation consumer — a **raw in-process Kafka consumer** off `studio-spine` (ADR-0013's canon-tier exception to Principle 2), applying `token.revoked` broadcasts to the deny-list evict-before-serve. And it must honor the **day-1 cell contract** (`JGAUQRsw2g`): `CELL_ID` distinct from `CLUSTER_NAME`, a `/healthz` echoing both, a cell-scoped topic, and an `orvexcell` fail-closed check on every consumed CloudEvent.

This epic exists because correctness must never depend on a cache (cold cache = slower, never wrong) and because the cell contract is load-bearing before a second cell — without the `orvexcell` fail-closed check a consumer could cross-consume another cell's revocation topic (Arch §1 named gap).
