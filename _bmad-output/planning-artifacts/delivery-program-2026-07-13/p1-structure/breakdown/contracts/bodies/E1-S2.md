## 🎯 Story
As a family of six+ consuming repos, I want a self-validation CI (`.github/workflows/validate.yaml`) that lints every OpenAPI 3.1 / JSON-Schema 2020-12 artifact, resolves every cross-reference, checks fixture well-formedness, enforces the T4 `version`-required rule, and enforces the T6 `$id`/`servers` hygiene, so that a malformed contract can never merge and break all consumers at once.

**Definition of Done:** ONE named test `TestValidateCatchesMalformedContract` — a CI-layer test that feeds a deliberately-broken artifact set (dangling `$ref`, content-event schema missing `version`, non-templated `servers:` literal) through the `validate.yaml` runner and asserts it exits non-zero naming each violation. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2091); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria
- [ ] **AC1** — Given a well-formed contract set, When `validate.yaml` runs, Then it passes: OpenAPI 3.1 lint + JSON-Schema 2020-12 lint + every `$ref`/`dataschema` resolves + fixtures well-formed. *Assert: exit code 0* [Source: Architecture §1 workload-topology, T5].
- [ ] **AC2** — Given a content-bearing / `profile: thin` event schema missing `version` in `required`, When CI runs, Then it fails. *Assert: exit ≠ 0, message cites the schema + `version`* [Source: SE-Arch T4].
- [ ] **AC3** — Given a schema with a bare-GitHub-web `$id` relied on for dereference, When the cross-ref check runs, Then it fails (no consumer may rely on `$id` dereferencing). *Assert: exit ≠ 0* [Source: SE-Arch T6].
- [ ] **AC4** — Given a non-templated `orvex.dev` literal in an OpenAPI `servers:` block, When CI runs, Then it fails (the cell-lint blind spot closed). *Assert: exit ≠ 0; templated `{cell}` form allowed* [Source: SE-Arch T6].
- [ ] **AC5** — Given a dangling `$ref` (negative path), When CI runs, Then it fails naming the unresolved ref. *Assert: exit ≠ 0* [Source: Architecture §1].
