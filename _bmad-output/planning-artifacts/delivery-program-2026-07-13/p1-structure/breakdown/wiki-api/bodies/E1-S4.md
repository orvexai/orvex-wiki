## 🎯 Story

**Audit: already implemented** (evidence in §4) — this story is VERIFY + harden: prove the behaviour via the named DoD test + adversarial review; do not rebuild.
As a **scoped agent caller**, I want **wiki-api to verify my identity-minted token at its edge via `orvex-studio-lib/auth` and forward it verbatim on every upstream call, holding no elevated service credential**, so that **I can never do through wiki-api what I couldn't do directly against the engine or knowledge**.

**Definition of Done:** ONE named test `TestTokenPassThroughNoAmplification` (integration — a scoped token that the engine would 403 is forwarded unchanged and the caller gets 403; wiki-api never substitutes a service credential). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2104); this story is dispatch-blocked until that tag exists.*

- [ ] **AC4 (negative/security)** — Given any code path, When it would exchange a caller token for an engine credential, Then it does not exist. *Assert: static/grep gate finds no credential-exchange path.* [Source: FR-A16, Arch A-TOKEN]
