## 🗺️ Epic — Response shaping layer & the contracts-authored descriptor

Two cross-cutting concerns that make the grammar usable and stable. **Shaping** is one shared middleware/interceptor layer — field projection (`fields=`), `concise|detailed` verbosity, cursor pagination, and body-offload (`resource_link` handles + token estimates) — applied to every verb, never re-implemented per route; it is what defends the ≤ ~1k-token KPI (G5) [Source: FR-A12, FR-A13, Arch A-SHAPE]. The **descriptor** is authored in orvex-studio-contracts (`openapi/wiki-api.yaml`, per the family authored-never-generated rule); wiki-api conforms and CI-diffs its registry-generated `/v1/openapi.json` against the authored contract as a drift-gate — a conformance input, never the codegen source [Source: FR-A14, Arch A-DESCRIPTOR].

Why it exists: MCP and the CLI codegen typed clients from the **contracts tag**, which also keeps anything AGPL-originated out of the closed codegen path (NFR-A7). Fail-loud is a contract: unknown routes and unknown body fields are rejected, never dropped on the floor. The authored contract is the compatibility contract — additive-only within a version, breaking changes only via a new version (NFR-A6). Body-offload handles dereference through the same ACL-enforced read path, so shaping introduces no existence oracles (FR-A17) [Source: FR-A13, FR-A14, NFR-A6].
