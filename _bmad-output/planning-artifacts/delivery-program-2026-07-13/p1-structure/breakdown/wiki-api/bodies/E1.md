## 🗺️ Epic — Phase-0 dual-surface strangler facade & Go runtime foundation

wiki-api exists so the wiki's agent contract stops living inside the AGPL engine fork. This epic stands up the Go service that terminates that contract: the Phase-0 **dual-surface facade** (D-A9) — a byte-compatible reverse proxy over the engine's already-live `/api/orvex/*` paths served *alongside* a draft `/v1` verb-grammar surface — plus the runtime foundation (six-tier layout per CS §6, config, health, edge auth, CI gates) every later epic builds on. It is deliberately an **extraction, not a construction**: everything heavy is already live in the engine, so Phase 0 is a thin authenticated proxy that MCP's deployed 322-path client repoints to with a genuine base-URL change (G4), making the engine-vs-wiki-api black-box parity gate meaningful [Source: FR-A15, Arch A-STRANGLER].

Why it exists: the fork is already 316 modified files / 121-file conflict surface and *is* the agent contract; moving the contract to a stable facade first is the strangler's opening move (G2) [Source: PRD §1]. wiki-api holds **no elevated credential** and makes **no authorization decision** — a scoped caller can never be amplified through it (the Phase-3 SSE-403 lesson) [Source: FR-A16, FR-A17].
