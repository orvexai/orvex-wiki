## 🎯 Story
**Audit: already implemented** (evidence in §4) — this story is VERIFY + harden: prove the behaviour via the named DoD test + adversarial review; do not rebuild.

As the indexer I want to extract attachment text via Tika on `wiki.attachment.created` (and re-extract on replace) so that document attachments are searchable, with a metadata-only fallback that never silently drops a file.

**Definition of Done:** `TestAttachmentTikaExtractionAndMetadataOnlyFallback` (integration, indexer + Tika helper) — an allow-listed, in-size-cap attachment is Tika-extracted and indexed; an unparseable or oversized file indexes metadata only and records a per-document extraction-failure state (never a silent drop). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2099); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria
- [ ] **AC1** — Given `wiki.attachment.created` with an allow-listed MIME under the size cap, When ingested, Then Tika extracts text and the attachment document is indexed with `source_type=wiki-attachment`. *Assert: extracted text queryable.* [Source: FR-K5]
- [ ] **AC2** — Given a replace, When `attachment.created` fires again, Then the attachment is re-extracted (version-guarded). *Assert: new content replaces old under CAS.* [Source: FR-K5]
- [ ] **AC3** (fallback) — Given an unparseable OR oversized file, When ingested, Then only metadata is indexed and `extraction_state=failed` is recorded — never a silent drop. *Assert: metadata present; extraction_state set; failure visible to staleness SLI.* [Source: FR-K5, FR-K28]
- [ ] **AC4** (negative) — Given a MIME outside the allow-list, When ingested, Then extraction is skipped (metadata-only) and no Tika call is made. *Assert: Tika call count = 0.* [Source: FR-K5]
- [ ] **AC5** (security) — Given the outbound fetch to the engine for the attachment blob, When fetching, Then it is SSRF-safe (pinned/allow-listed host, cluster-local). *Assert: non-allow-listed target refused.* [Source: FR-K5, NFR-K8]
