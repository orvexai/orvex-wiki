## 📖 Epic — Studio corpora & Memory retrieval

Per R-9 (Decision 4, binding): knowledge is **THE one general multi-corpus search** — the Studio skills-RRF stack, Memory recall, and chat-history recall all retire INTO it. This epic indexes the Studio source events `studio.skill.*`, `memory.*`, `chat.*` on the studio-spine broker as their own corpora under the FR-K7 multi-source model (FR-K31b), enforces the **A12 confidentiality controls** (owner-scoped admission, employer/org-admin firewall, delegated-principal-only, ingestion-side privacy exclusion, per-corpus fail-closed default), builds the **Memory retrieval namespace wall** (AD-7 — owner_scope-keyed Turbopuffer namespace + AD-11 per-card ordering & read-your-writes), and owns the **retrieval eval harness** (FR-E1..E4, AD-9 — recall@10 ≥ 0.80, regression gate, raw wiki stays searchable alongside distilled Memory).

Why it exists: this is what makes "your AI finally knows you" retrievable and safe. A user's Memory and chat history are **invisible to their employer by construction** — no org role, including admin, admits a non-owner. Per R-9 D-R9-1/2 (BINDING): Studio publishers emit ready-made chunks and each corpus pins its own embedding model; cross-corpus fusion stays correct because the query layer fuses by RANK (RRF), never raw score.
