## 🎯 Story
As the wiki search box I want API-path saves to be keyword-findable at save-ack via a synchronous knowledge upsert hook so that users get read-your-writes without waiting for the async event path.

**Definition of Done:** `TestSyncKeywordUpsertReadYourWritesAndBreaker` (integration, `internal/workflow` + `/internal/v1/keyword-upsert`) — a post-commit-pre-ack call (budget ≤ 50 ms, mTLS + (tenant,page-id) coherence + version bound) makes the page keyword-findable immediately; when knowledge is unreachable the call breaks and the save path degrades to async (the save NEVER fails). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2099); this story is dispatch-blocked until that tag exists.*
