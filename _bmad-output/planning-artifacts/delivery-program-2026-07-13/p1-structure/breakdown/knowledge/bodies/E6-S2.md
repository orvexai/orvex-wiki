## 🎯 Story
As the bundle egress guard I want bundles to be principal-bound capabilities built as point-in-time ACL snapshots so that `GET /v1/bundles/:id` cannot be fetched by another principal (closing the IDOR), and delivery is never a bare presigned URL.

**Definition of Done:** `TestBundleIsPrincipalBoundCapabilityNoIDOR` (integration, `cmd/bundler`) — a bundle stored on S3 with a random id is served only when `principal == creator AND tenant match`; a cross-principal fetch is rejected; delivery streams through the authenticated gateway (or a presigned URL minted only after the ownership check). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2099); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria
- [ ] **AC1** — Given a built bundle, When stored, Then `bundle_registry` records `{bundle_id(random), creator_principal, tenant, org_or_realm, expiry}` (IdP-agnostic). *Assert: registry row created with random id.* [Source: FR-K19]
- [ ] **AC2** — Given `GET /v1/bundles/:id`, When fetched, Then it enforces `principal == creator AND tenant match` before serving. *Assert: owner fetch succeeds.* [Source: FR-K19]
- [ ] **AC3** (IDOR) — Given a different principal with a valid token, When fetching another's bundle id, Then it is rejected. *Assert: cross-principal fetch → 403; in NFR-K3 probe set.* [Source: FR-K19, NFR-K3]
- [ ] **AC4** — Given delivery, When served, Then it is NEVER a bare presigned S3 URL — it streams through the authenticated gateway or a short-lived presigned URL minted only after the ownership check. *Assert: no bearer-capability URL issued pre-check.* [Source: FR-K19]
- [ ] **AC5** — Given a point-in-time snapshot, When fetched later, Then it is NOT re-filtered on fetch (owner-binding is the safeguard) and expires per short lifecycle. *Assert: snapshot semantics + expiry enforced.* [Source: FR-K19]
