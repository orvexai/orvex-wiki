## 🎯 Story
As the shipped CLI I want the SSE gateway's load-bearing deltas — `/events/head`, numeric-monotonic epoch-ms cursor, legacy unprefixed event names, `content_updated` pageIds-only, 1 MiB frames, and open-status backoff — so that cold start, cursor advance, lag metrics, and frame parsing all keep working.

**Definition of Done:** `TestSSECLIDeltasCursorNamesAndBackoff` (integration, `cmd/sse`) — `/events/head` returns `{stream_head}`, the cursor is numerically monotonic + epoch-ms-derivable (or each frame carries a server epoch-ms), the gateway emits legacy unprefixed names, `content_updated` carries `{pageIds}` and no body, frames cap at 1 MiB, heartbeat < 45 s, and 401/403/503/404/429 open-statuses map to the right client backoff. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2099); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria
- [ ] **AC1** — Given cold start, When `GET /events/head` is called, Then it returns `{"stream_head":"<id>"|null}` to seed `Last-Event-ID`. *Assert: head returned; SSE opens with a resume point.* [Source: FR-K15a]
- [ ] **AC2** (cursor) — Given the SSE `id`, When the client advances, Then the cursor is numerically monotonic (smaller-or-equal = no-op, crash-safe) AND epoch-ms-derivable for lag — either `<ms>-<seq>` ids or a separate server epoch-ms per frame. *Assert: monotonic compare + lag derivation both work; A6a decision honored.* [Source: FR-K15a]
- [ ] **AC3** (names) — Given the CLI-facing stream, When frames are emitted, Then they use legacy unprefixed names (`page.updated`/`page.content_updated`/`comment.*`/`attachment.*`/`space.*`/`permission.changed`), NOT `wiki.*`. *Assert: a `wiki.*` name never reaches the client.* [Source: FR-K15a]
- [ ] **AC4** — Given `content_updated`, When emitted, Then it carries `{pageIds:[…]}` (or single `id`) and NO body — the CLI refetches each body (FR-K20). *Assert: no body in frame; pageIds present.* [Source: FR-K15a]
- [ ] **AC5** (limits + backoff) — Given frame size/heartbeat/open-status, When served, Then max frame 1 MiB, heartbeat < 45 s, and 401/403→5 min auth backoff, 503/404→polling honouring `Retry-After`, 429→`Retry-After`. *Assert: each status maps to the right backoff.* [Source: FR-K15a]
