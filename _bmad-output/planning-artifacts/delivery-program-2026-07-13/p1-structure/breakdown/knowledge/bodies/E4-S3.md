As the auth boundary I want an IdP-agnostic verifier (JWKS session tokens + Orvex api-keys, via orvex-studio-lib) that yields the polymorphic {user|org} principal so that knowledge never talks to an IdP directly, never runs anonymous, and never uses a shared APP_SECRET.
