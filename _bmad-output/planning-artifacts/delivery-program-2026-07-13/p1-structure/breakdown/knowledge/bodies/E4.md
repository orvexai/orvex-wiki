## 📖 Epic — Universal ACL choke point, auth & isolation

The single security spine of the service: **one ACL choke point that governs every content-bearing egress surface** (FR-K12) — F3 queries, SSE frames, bundles, projection download. Every egress narrows through the engine's FR-13 ACL primitives ∩ api-key token scopes (C5) for the verified principal, under two fixed invariants: staleness may only over-narrow, never over-expose; engine-down ⇒ fail closed. This epic also builds the auth verifier (FR-K13, A5a — IdP-agnostic via identity, polymorphic {user|org} principal), the FR-13 hot-path overload protection (A1a: single-flight + interactive/bulk bulkhead + rate contract), and the automated isolation probes (NFR-K3) that run in CI and against every deploy.

Why it exists: G5 — cross-tenant result bytes = 0 AND intra-tenant restricted-page bytes = 0, enforced structurally + fail-closed. This is the invariant the whole product's trust rests on. A1a keeps the shared engine FR-13 endpoint from becoming a self-inflicted single point of overload; A5a keeps auth IdP-agnostic (Clerk or Keycloak) so knowledge never talks to an IdP directly.
