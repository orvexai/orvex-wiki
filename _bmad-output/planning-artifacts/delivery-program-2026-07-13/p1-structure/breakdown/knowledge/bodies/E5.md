## 📖 Epic — SSE gateway & Orvex CLI wire contract

knowledge relocates the engine's events controller and must preserve the **full shipped SSE wire contract** the Orvex CLI (formerly docmost-cli) and orvex-docs daemons depend on (FR-K15/K15a, A6/A6a). A KafkaSink bridges rich CloudEvents onto a single-partition raw topic (`wiki-events.<cell>`) so offset cursors are total-order-correct; each SSE pod runs a standalone assigned consumer (assign/seek, no shared group). The gateway serves replay-then-tail with `Last-Event-ID` resume, heartbeats, and control frames — and it must emit the **legacy unprefixed event names** (not `wiki.*`) or the CLI skips every frame as unknown.

Why it exists: the CLI's cache freshness silently breaks if any wire detail regresses. FR-K15a enumerates the load-bearing deltas: `/events/head` cold-start bootstrap, `<ms>-<seq>` cursor semantics (monotonic compare + epoch-ms lag), `content_updated` carrying `{pageIds}` only, 1 MiB frames, 401/403 + 503/404 backoff, <45 s heartbeat. The A6a cursor-scheme tension is an ADR trigger to be pinned in contracts before CUTOVER.
