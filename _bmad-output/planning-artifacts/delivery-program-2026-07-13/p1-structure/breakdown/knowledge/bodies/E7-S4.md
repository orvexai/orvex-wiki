## 🎯 Story
As the Orvex CLI I want the read verbs (search, ai-ask retrieval, related, semantic) to call the knowledge query API directly with an offline fallback to the local cache, and a projection-parity guarantee so that the projection download agrees with index results at watermark ≥ cursor.

**Definition of Done:** `TestCLIReadVerbsDirectAndProjectionParity` (integration, query + projection) — CLI read verbs hit `/v1/query` directly (offline → local cache), and at watermark ≥ event cursor the projection download agrees with index results (documents sync-upserted via FR-K4 ahead of their watermark are excluded from the parity claim until the event lands). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2099); this story is dispatch-blocked until that tag exists.*

- [ ] **AC1** — Given CLI read verbs (`search`, `related`, semantic, and `ai ask` retrieval sub-call), When online, Then they call the knowledge query API directly. *Assert: verbs hit /v1/query (ask routes to ai per D-A12; knowledge serves the retrieval sub-call).* [Source: FR-K21]
- [ ] **AC2** (offline) — Given no network, When a read verb runs, Then it falls back to the local SQLite cache. *Assert: offline read served from cache.* [Source: FR-K21]
- [ ] **AC3** (parity) — Given watermark ≥ event cursor, When compared, Then the projection download agrees with index results. *Assert: projection ≡ index at watermark.* [Source: FR-K22]
