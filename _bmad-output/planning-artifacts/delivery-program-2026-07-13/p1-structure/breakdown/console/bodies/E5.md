## 🗺️ Feature area

The cross-satellite admin operations that let an operator run routine ops **without kubectl/psql/SSH** (G2) — built to the A-ADMIN rule: *"console calls the owning satellite's authenticated admin API; the satellite keeps its own authz and audit — console adds its gate and its audit record on top, never a bypass."* This epic covers the identity- and knowledge-fronted surfaces: org/tenant registry browse with provisioning history + lifecycle-state inspection (FR-CT7 — the write half is registry-browse only until identity commits an operator-lifecycle admin surface, OQ-CT9; FR-I14 is workflows-only and *expressly not callable by console*); token management — list/inspect/rotate/revoke MCP/agent/api tokens family-wide filtered by tenant/subject/scope, where a revocation *is* identity's revocation (FR-CT8); and satellite operations — knowledge reindex/rebuild triggers via knowledge's FR-K25 admin surface (knowledge then enqueues its own spine CloudEvent per FR-W10 — *console is never a spine publisher for another domain*), the uniform wrap other satellites' admin actions grow into (FR-CT9).

It exists because the admin side is scattered across identity (FR-I7/I8/I17), knowledge (FR-K25), and others — console is where those terminate as one audited, composed surface.
