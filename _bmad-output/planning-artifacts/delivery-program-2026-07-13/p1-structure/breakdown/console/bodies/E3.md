## 🗺️ Feature area

Half 1 of the console charter: **one authenticated pane over the LGTM stack** so an operator can follow a single incident logs ↔ metrics ↔ traces ↔ workflow-history without leaving console (G1). This epic builds the guardrailed BFF proxy (`/api/obs/{logs,metrics,traces,alerts}`) with the FR-CT6 counter-metric enforced server-side (bounded ranges, cardinality/volume caps, cancellable queries, per-operator rate buckets) so *"console is never the reason Mimir/Loki fall over during the incident an operator is diagnosing"*; the three signal panes built the DECIDED way (D-OBS-UI / OQ-CT1 resolved): Grafana's Apache-2.0 `@grafana/ui` + `@grafana/scenes` supply the pane **shell** (layout, panels, query editors — the beautiful Grafana controls), while the **trace waterfall and the LogQL/TraceQL result panes are ORIGINAL console code** because Grafana's trace-viewer is AGPL-only and `@grafana/ui` ships zero trace components (NFR-CT4 intact); the cross-signal correlation pivots keyed on the contracts FR-C18 label vocabulary (FR-CT2); the alert-state surface (FR-CT5); and the satellite health board rendering each sibling PRD's named SLIs from Mimir (FR-CT4 — console *re-renders, never redefines*).

It exists because every satellite already points its telemetry the same way *"consumed by orvex-studio-console"* — console is where those threads terminate in one usable pane.
