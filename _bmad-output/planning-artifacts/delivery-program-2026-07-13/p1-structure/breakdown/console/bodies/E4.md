## 🗺️ Feature area

Temporal workflow visibility and the family's most dangerous buttons — retry/replay, cancel/terminate, signal — all reached **by removal, not confinement** (D-CTRL, *"the sharpest decision in the doc"*): console holds **no direct Temporal credential**, grants no `temporal-frontend` egress, and every read and write goes through the **thin authorizing proxy orvex-studio-workflows owns** in front of the unauthenticated OSS frontend. The proxy verifies the operator's pass-through token and enforces workflows-side who-may-signal authz (OQ-W8). This epic builds console's single Temporal client interface bound only to that proxy: workflow list/filter/inspect reads across the family's domains (clerk-lifecycle, knowledge rebuild, ai tool-loop, org-conversion) with status/history/pending-activities/retry/schedule state (FR-CT3); the write ops each gated on fresh no-grace introspection + typed confirmation (FR-CT10); and the workspace/user lifecycle state-machine view end-to-end (Clerk webhook → workflow → engine + studio legs) including stuck/failed workflows and reconcile drift reports so *"is this org actually deprovisioned everywhere?"* is answerable in one screen (FR-CT11).

It exists because the workflows arch explicitly punts operator UI (*"Temporal Web UI … not this repo"*) — console is that surface, gated so D-WF-1 one-Temporal-home ownership stays intact.
