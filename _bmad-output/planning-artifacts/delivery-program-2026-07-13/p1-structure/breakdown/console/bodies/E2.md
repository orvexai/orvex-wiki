## 🗺️ Feature area

Console is *"the family's highest-privilege UI"* — the BFF is **a hard privilege boundary** because LGTM cannot verify identity-minted tokens and trusts console's edge entirely (Arch A-EDGE, NFR-CT2). This epic makes authN/Z, credential separation, and audit **structural, not per-route conventions**: every route registers through the `lib/auth` router deny-by-default (undeclared route rejected at startup); operator principal = Clerk admin-org membership verified via identity; every mutation additionally requires fresh, no-grace introspection (`RequireFresh`) plus a typed confirmation; satellite admin calls pass the **operator's token through** so the owning satellite re-verifies (composition adds a gate, never removes one — the wiki-api FR-A16 lesson); the only standing service credential is the read-only LGTM one, confined to the observability package (no Temporal credential exists at all, D-CTRL). Every admin mutation is wrapped intent-audit → upstream call → outcome-audit, emitted to the durable, ackable **family audit stream** (Kafka topic on `studio-spine`) — *audit failure fails the mutation* (FR-CT14). The audit-trail browse pane over that stream (FR-CT12) closes the loop.

It exists because *"one compromised console session reaches every satellite's admin plane plus all logs"* (Arch §5) — the concentrated-privilege risk the PRD counter-metrics name; this epic is the mitigation.
