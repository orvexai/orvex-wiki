## 🗺️ Feature area

The money-and-limits panes — where console **displays but never owns**. Per the 2026-07-05 C3 ruling (D-CAP re-pointed): **orvex-studio-billing owns the plan→entitlement→AI-spend-cap VALUE (system of record, D-P3)**; ai READS that entitlement and ENFORCES it via the LiteLLM virtual key's `max_budget` + per-caller scoped keys; **console displays both and stores neither**. This epic builds: the AI usage & enforcement pane over ai's FR-AI15 reporting API — per-tenant spend vs cap, spend by feature, historical usage, cap-denial counts, spend-check staleness, per-caller scoped-key budgets, with `ai.cap.*` events surfaced via the alert-state surface (not a console-owned spine subscription) (FR-CT12a); and the plan/entitlement + quota-usage pane — each org's plan (Free / £7 Personal / Teams / Enterprise) and entitlement set from billing, alongside live quota usage read back from the engine's F-QUOTA counters (pages/bytes/files/members — Free ceiling 200 pages / 1 GiB / 2,000 files / 25 members; £7 Personal 20,000 / 50 GiB / 20,000 / 25), fronting billing's entitlement-admin surface for the plan/entitlement/cap-value *set* (Phase 3) under the uniform FR-CT9 wrap (FR-CT12b).

It exists so an operator can answer *"what plan is this org on, what's it using, and is it near its cap?"* in one place — without console becoming a second source of truth (the standing BFF-creep failure mode).
