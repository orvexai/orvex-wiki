## 🗺️ Feature area

Console was created after the scaffolding wave and is an **empty scaffold** (4 files, no `go.mod`) — so before any pane or admin surface can ship it must become a real, cell-contract-onboarded Go service. This epic lands the foundation the whole service is built on: the single `cmd/console` binary that serves both the embedded SPA and `/api/*`, the family's **first non-pure-Go build** (a node SPA build feeding the `go:embed` path, A-FRONTEND / D-CT1), the CI gates that make console safe to run (AGPL-import guard, the LGTM-credential-confinement build-tag/lint gate), the cell-contract onboarding (cell-lint, `CELL_ID`/`CLUSTER_NAME`, cell-guard middleware, the tenant-move typed 501 stub), the always-warm Deployment an ops pane requires (NFR-CT3), the endpoint-as-config posture that keeps the cross-cell decision cheap (NFR-CT6/OQ-CT4), and the Redis-only stateless store + the honest per-backend rendering framework (FR-CT17/CT18/NFR-CT7, A-STATE / A-HONESTY).

It exists because console *"sits outside the day-1 cell-contract ruleset until the bootstrap Issue lands"* (Arch §5, SE-Arch F5) — an honestly-disclosed bootstrap gap that blocks everything else. Every sibling epic assumes this platform is in place.
