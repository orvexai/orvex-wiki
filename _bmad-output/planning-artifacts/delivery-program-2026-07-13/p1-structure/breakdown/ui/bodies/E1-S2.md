## 🎯 Story

As a **build agent**, I want **a typed `/api` client generated from the `orvex-studio-contracts` codegen plus the identity-exchanged, cell-bearing session-token attach** so that **every surface consumes exact contract types and the Cloudflare Worker can route by the `cell` claim at the edge** — the scaffold ships no data-fetch lib and no contracts dep.

**Definition of Done:** ONE named test `TestApiClientAttachesCellToken` — an integration test that drives a client call through a faked BFF and asserts the outbound request carries the identity-exchanged session token (with a `cell` claim), NOT the raw Clerk JWT, verified through the request interface. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2109); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given the pinned contracts tag, When codegen runs, Then TS DTO + SSE-grammar types are emitted and `tsc -b` passes against them. *Assertion: generated types compile; a drift from the tag fails `tsc`.* [Source: A-STREAM, NFR-UI5, CS §7]
- [ ] **AC2** — Given a signed-in session, When any `/api` request is made, Then the identity-exchanged cell-bearing token is attached. *Assertion: `Authorization` header token decodes to include a `cell` claim; a raw Clerk JWT (no `cell`) is never attached.* [Source: A-STATELESS]
- [ ] **AC3** — Given the client, When it targets any host, Then only same-origin `/api` is used (Clerk SDK + Stripe redirect allowlisted). *Assertion: no other origin appears; egress guard green.* [Source: A-SHELL, NFR-UI5]
- [ ] **AC4 (error)** — Given a 401/expired token, When a call is made, Then the client surfaces a typed auth-error frame (no silent retry-loop, no token minting). *Assertion: error is typed; the SPA never mints/persists a token.* [Source: A-STATELESS]
- [ ] **AC5 (negative)** — Given anon, When a public `/api` call is made, Then no session token is attached and the call still resolves. *Assertion: header absent; response typed.* [Source: FR-UI2 R5]
