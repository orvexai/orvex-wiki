## ✅ Acceptance Criteria

- [ ] **AC1** — Given any AI-originated artifact, When it renders, Then it shows as a proposal with an explicit accept affordance and a provenance label. *Assertion: role/label exposes "proposal"; a provenance tag is present.* [Source: FR-UI27, A-DRAFT]
- [ ] **AC2** — Given a proposal, When accept has not been recorded, Then it never renders as committed. *Assertion: committed styling/state absent until a BFF-confirmed accept event.* [Source: FR-UI27, G3]
- [ ] **AC3** — Given an accept, When the BFF confirms, Then the artifact flips to committed once and provenance persists. *Assertion: single state transition on the confirmation frame.* [Source: FR-UI27]
- [ ] **AC4 (error)** — Given the accept round-trip fails, When the error returns, Then the proposal remains pending with inline retry (no half-committed visual). *Assertion: state stays proposed; retry present; NFR-UI6 no half-commit.* [Source: FR-UI27, NFR-UI6]
- [ ] **AC5 (negative)** — Given a severed stream, When the connection drops mid-proposal, Then the pending proposal simply remains pending. *Assertion: no auto-commit on disconnect.* [Source: NFR-UI6]
