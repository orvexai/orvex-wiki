## 🎯 Story

As a **signed-in user**, I want **the app shell — a left sidebar with 6 destinations, the single ⊕ Create CTA, the account footer menu, a ⌘K palette, an inbox drawer, and a responsive desktop/tablet/mobile layout** so that **I can navigate the whole Studio from one consistent frame** (FR-UI1, A-SHELL).

**Definition of Done:** ONE named test `TestShellRendersSixDestinationsAndCreateCta` — a component-integration test asserting the sidebar renders exactly the 6 destinations (Home/Discover/Your Skills/Your Memory/Your Wiki/Review), the ⊕ Create as an action (not a 7th nav row), and the mobile bottom-tab shell at <768px, verified through the rendered nav landmark. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2109); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a signed-in session, When the shell mounts, Then the sidebar shows the 6 destinations with routes `/app`, `/app/discover`, `/app/library`, `/app/memory`, `/app/wiki`, `/app/review` and the ⊕ Create CTA as a dominant action. *Assertion: 6 nav items + 1 action; Create is not a nav row.* [Source: FR-UI1]
- [ ] **AC2** — Given the shell, When ⌘K is pressed, Then a focus-trapped palette opens mirroring nav, keyboard-complete, with a no-match empty state and context-gated actions (never global no-ops). *Assertion: focus trapped; no-match state; disabled context actions.* [Source: FR-UI1, NFR-UI2]
- [ ] **AC3** — Given the inbox drawer (desktop), When there are unread+actionable items, Then the badge is a number pill using text+shape (never color-alone) and is absent at zero. *Assertion: badge text+shape; absent at 0.* [Source: FR-UI1, NFR-UI2]
- [ ] **AC4** — Given breakpoints, When width changes, Then desktop ≥1024 persistent sidebar collapses to a 64px rail (state persists per user), tablet 768–1023 is an icon-rail overlay, mobile <768 is a bottom tab bar (Home/Discover/Create/Review/"You" sheet). *Assertion: correct layout per breakpoint; all targets ≥48×48px.* [Source: FR-UI1, NFR-UI8]
- [ ] **AC5 (negative)** — Given a personal (non-Teams) user, When the shell renders, Then there is zero org chrome (no org-switcher/org-context). *Assertion: no org UI present.* [Source: FR-UI1, D-S17]
