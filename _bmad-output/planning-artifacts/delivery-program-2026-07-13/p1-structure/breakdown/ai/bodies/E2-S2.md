## 🎯 Story

As a page-edit tool (`insert-at-cursor`, `update-block-by-id`, batch edits), I want to execute every mutation as an orvex-wiki-api block-chokepoint call with the caller's token, so that ai holds no elevated write credential and a scoped caller can never write through ai what it couldn't write directly (FR-AI9).

**Definition of Done:** one named test `TestPageEditGoesThroughChokepointWithCallerToken` (integration) runs a page-edit tool and asserts the write is a wiki-api chokepoint call carrying the caller's token, with `ifVersion` CAS and a read-after-write receipt returned — never a direct DB or ydoc path. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2097); this story is dispatch-blocked until that tag exists.*
