## 🗺️ Feature area

LiteLLM is the only model gateway family-wide (D-AI3) — every chat/completion/tool-round/synthesis/image call leaves the fork through it, which closes FR-39. This epic covers the gateway port/adapter, per-tenant model configuration (the `workspace_ai_credentials` successor minus embeddings), the FR-AI19a write-mask that makes base-URL/virtual-key read-only (so a tenant can't re-point spend off the capped key), the PII/secret hygiene middleware that is the last hop before LiteLLM, and image generation (paid-gated).

**Why it exists:** no in-fork model path (G1, FR-39) and no second gateway — a tenant-writable key field would silently defeat the FR-20 backstop (FR-AI19a, D-AI9). PII must fail closed on the strictest policy when policy lookup fails (FR-AI20; Arch A-PII).
