## 🎯 Story

As the enforcer, I want to read the tenant's plan→cap entitlement from billing (SoR, D-P3) — subscribing to `billing.entitlement.changed` on the Kafka spine (short-TTL cache, evicted on that event; a billing entitlement read on cache miss) — and apply it to the per-caller budgets + per-tenant `max_budget` via a reconcile schedule, so that ai enforces but never stores the cap value (FR-AI12).

**Definition of Done:** one named test `TestEntitlementChangePropagates` (integration) publishes a `billing.entitlement.changed`, asserts the Redis entitlement cache is evicted, and asserts the per-caller budgets + `max_budget` are re-asserted from the new entitlement within one reconcile interval. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2097); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a cache miss, When enforcement needs the entitlement, Then ai reads it from billing and caches it short-TTL. *Assert: billing read on miss; cached.* [Source: FR-AI12]
- [ ] **AC2** — Given `billing.entitlement.changed`, When consumed, Then the Redis entitlement cache is evicted and budgets re-asserted within one reconcile interval. *Assert: propagation ≤ reconcile interval (NFR-AI4 bound).* [Source: FR-AI12, NFR-AI4]
