## 🗺️ Feature area

The FR-20 cost-protection spine — the P0 that must be stood up before any feature (Rollout step 1). Enforcement is layered (A-METER): per-caller scoped LiteLLM keys + budgets (call-site ceiling, D-S5) over a per-tenant `max_budget` backstop that holds even when ai's own metering is buggy. The cap **value** is billing's entitlement (SoR, D-P3/D-AI11): ai reads it via `billing.entitlement.changed` + entitlement read, and enforces it. Free = the ~zero-cost model-class allowlist (D-AI12) — frontier denied 402, zero-cost uncounted; £7 Personal = a spend cap. Usage journals to Postgres append tables (Mongo struck, D-S12) via a Kafka consumer; embeddings ride a separate nested budget (D-S15).

**Why it exists:** cost can never run away — a runaway loop, compromised token, or buggy client cannot exceed the cap beyond the enforcement-lag bound (G2, NFR-AI4). The named regression: a spend-lookup failure must NEVER read as zero spend.
