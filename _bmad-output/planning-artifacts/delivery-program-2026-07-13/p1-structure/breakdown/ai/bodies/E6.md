## 🗺️ Feature area

Interactive turns run in-request; anything long (multi-page drafts, corpus-scale jobs, tools-retry escalation, maintenance sweeps) is a central Temporal workflow in orvex-studio-workflows calling ai's idempotent step-APIs (D-WF-1). ai runs NO workflow worker. This epic authors the step-API shape that workflows FR-W11/OQ-W4 is blocked on: `model-call`, `tool-exec`, `result-sink` — all idempotent (lib FR-L19), `WorkloadIdentityOnly` (lib FR-L6), metered like any request. Caller authority rides as an identity-minted delegation token or opaque handle, never a raw token in workflow inputs (D-AI10).

**Why it exists:** durable where it matters (G5, FR-AI21/22) — long loops survive crash/restart with hard iteration/wall-clock/token budgets re-checked per step server-side. A durable loop cannot bypass FR-20 (Arch A-LOOP).
