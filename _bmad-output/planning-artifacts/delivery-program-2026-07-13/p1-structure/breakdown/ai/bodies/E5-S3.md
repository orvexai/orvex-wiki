## 🎯 Story

As the FR-20 backstop's integrity guard, I want the gateway base URL and tenant virtual key to be platform-provisioned and read-only post-cutover — the settings surface accepts the request shape but returns a typed rejection for writes to those fields — so that a tenant cannot re-point spend off the capped key (FR-AI19a, D-AI9; the named G3 exception).

**Definition of Done:** one named test `TestSettingsWriteMaskRejectsKeyRepoint` (integration) submits a settings write to base-URL/virtual-key and asserts a typed rejection (or documented ignore), while writes to model names / PII policy / soft-cap prefs succeed. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2097); this story is dispatch-blocked until that tag exists.*

- [ ] **AC2** — Given a write to model names / PII policy / soft-cap prefs, When submitted, Then it succeeds. *Assert: tenant-writable fields update.* [Source: FR-AI19a]
