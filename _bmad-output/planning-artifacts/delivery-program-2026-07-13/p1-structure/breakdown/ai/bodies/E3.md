## 🗺️ Feature area

The cited-ask/RAG loop is owned end-to-end by ai (D-ASK). `internal/ask` is a single package: retrieval is a model-callable tool, the loop runs agentically (multi-hop refinement, follow-up fetches, cross-checking), every hop calls knowledge `retrieve` with the caller's delegated principal (no ACL regression), and ai synthesizes the K5 shape `{answer, citations[], confidence, unanswered, gapNote, followups}`. Two front doors — the client `/api/ai/ask` and MCP's AI tools calling ai directly (ai as third MCP upstream) — run the identical core so answer quality can never fork (G4). Inline AI and the AI draft slot ride the same metering/PII pipeline.

**Why it exists:** no other service assembles cited answers — wiki-api's `ask` verb is removed (D-ASK). Grounding is non-negotiable: below-threshold support returns `unanswered` with a `gapNote`, never a fabricated answer (PRD §4 F2; Arch A-ASK; NFR-AI5).
