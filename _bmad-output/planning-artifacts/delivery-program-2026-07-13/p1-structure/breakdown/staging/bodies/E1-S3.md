## 🎯 Story

As a **staging platform engineer** I want **request authentication via `orvex-studio-lib/pkg/auth` Middleware with deny-by-default scope, the `app.tenant` GUC set from the verified principal, and `VerifyFresh` on high-privilege verbs** so that **the as-built trusted-header seam never recurs and every request is bound to a verified `{user|org}` tenant**.

**Definition of Done:** one named test `TestAuthMiddlewareDeniesUnverifiedAndSetsTenantGUC` (integration — a request with no/invalid token is 401; a valid identity-minted token sets `app.tenant` GUC to the verified principal; an apply/dial-change verb without `VerifyFresh` is rejected). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2094); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a request with no valid token, When it hits any verb, Then it is denied by default. *Assert: HTTP 401 and no handler code runs.* [Source: GOacy9kdJz P4+ADR-0009 deny-by-default]
- [ ] **AC2** — Given a valid token, When the handler runs, Then `app.tenant` GUC equals the verified principal (polymorphic `{user|org}`). *Assert: a DB query in-request sees the GUC set; body-supplied tenant is ignored.* [Source: GOacy9kdJz AD-8; ADR-0001/0012]
- [ ] **AC3** — Given a high-privilege verb (apply, dial change, pack approval), When invoked, Then `VerifyFresh` is required. *Assert: a stale-but-valid token is rejected on those verbs, accepted on read verbs.* [Source: GOacy9kdJz AD-8, P4]
- [ ] **AC4 (negative)** — Given a body-supplied principal/session id, When trust/provenance keys, Then the verified principal wins. *Assert: keying uses the token identity, never the body.* [Source: GOacy9kdJz AD-8]
