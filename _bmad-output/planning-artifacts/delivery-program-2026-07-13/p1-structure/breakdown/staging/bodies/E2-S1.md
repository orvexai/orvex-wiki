## 🎯 Story

As a **fleet agent** I want **to submit a Proposal in a single call — carrying one of the 5 Operations, a target ref/placement hint, DfM or ProseMirror-JSON content, rationale, and source-conversation ref — and get back a receipt with the proposal + ChangeSet id and a base-version stamp** so that **I can save a conversation's knowledge without ever touching the wiki, and later conflict detection has an anchor**.

**Definition of Done:** one named test `TestSubmitProposalStampsBaseVersionAndRejectsLossy` (integration — a valid submit returns a `SubmitReceipt` with proposal+ChangeSet ids; a page-targeting Proposal records the current wiki-api version token; markdown/HTML/lossy content is rejected at the door; the wiki DB is untouched). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2094); this story is dispatch-blocked until that tag exists.*
