## 🎯 Story

As a **staging platform engineer** I want **large Proposal bodies (and pre-mutate snapshots) stored by reference in Rook-Ceph S3 under `{tenant_id}/…` keys, with queue rows carrying only a pointer + sha256 content-hash** so that **100 pending Proposals cost storage only — no pinned compute — and S3 keys never leak tenancy across the wall**.

**Definition of Done:** one named test `TestPointerStatePayloadRoundTripsAndCascadesErase` (integration — a submit stores the body in S3 under the tenant prefix, the row holds pointer+sha256, a read reconstructs the exact bytes, and a GDPR erase cascades through payloads/snapshots leaving a content-free tombstone). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2094); this story is dispatch-blocked until that tag exists.*
