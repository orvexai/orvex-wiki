## 🎯 Feature area

`orvex-studio-staging` is one closed Go satellite in the six-tier family shape (AD-1): the Librarian is domain packages inside it, never a second deployable. Nothing in the PRD can be built until the satellite substrate exists — the state-machine pipeline (intake→triage→review→apply→sweep) advances rows only via CAS transitions inside a service that already has env-only config, `orvex-studio-lib/pkg/auth` middleware, an RLS-fail-closed Postgres store, the transactional outbox→relay→spine seam, and contracts codegen. This epic is the load-bearing floor every later epic stands on.

Why it exists: the current-state map records staging as **100% pre-build** and every satellite's recurring defect is a trusted-header auth seam, deferred RLS, and unbuilt lib. This epic front-loads exactly those (NFR-STG5 tenancy, NFR-STG6 observability, ADR-0009 auth, ADR-0011/cell-lint, ADR-0014 store, ADR-0007 events) so no story ever re-invents them.
