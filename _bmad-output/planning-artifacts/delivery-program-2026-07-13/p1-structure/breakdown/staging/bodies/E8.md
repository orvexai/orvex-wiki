## 🎯 Feature area

Staging is a platform capability with first-class MCP + CLI sections, and agent direct-write to the wiki is **removed, not deprecated-in-place** (no-fallbacks rule). This epic builds the public surfaces and the migration: new `staging_*` MCP tools (FR-STG23), the `orvex-cli staging` namespace with parity verbs (FR-STG24), `studio.staging.*` CloudEvents on the spine as telemetry — with `wiki.*` remaining the single reindex truth (FR-STG26), the hard-cut of ALL agent write surfaces enforced at the wiki-api ingress with loud migration errors and sequenced LAST (FR-STG25), and the verbatim Curator/Card-v1 supersession with golden-verified field mapping (FR-STG25, §9).

Why it exists: SM-1 requires 100% of agent knowledge writes to flow through staging post-cut — the cut plus the wiki-api 403 ingress make direct agent writes structurally impossible. The supersession absorbs `orvex-studio-api`'s Curator (a live double-ownership until cutover) into one classification path, not two.
