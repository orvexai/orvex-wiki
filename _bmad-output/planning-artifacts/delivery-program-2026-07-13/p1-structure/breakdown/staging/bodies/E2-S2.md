## 🎯 Story

As a **fleet agent (and the review surface)** I want **Proposals to group into a per-session ChangeSet whose lifecycle is an explicit state machine (`open → submitted → in-review → partially-accepted/accepted → applying → applied / rolled-back / rejected / expired`) advanced only via CAS transitions, submittable explicitly or by idle auto-submit** so that **a session's work is one reviewable, schedulable, rollback-able unit and no two writers can diverge its state**.

**Definition of Done:** one named test `TestChangeSetLifecycleCASTransitionsOnly` (integration — legal transitions succeed via `lifecycle` CAS; an illegal transition is rejected; a concurrent double-advance loses the CAS; idle timeout auto-submits an `open` ChangeSet). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2094); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given an open ChangeSet, When an agent submits explicitly, Then it moves `open→submitted`. *Assert: state column = submitted; an outbox event emitted in the same tx.* [Source: mOzNDhn322 FR-STG2; GOacy9kdJz AD-2]
- [ ] **AC2** — Given an idle `open` ChangeSet past the timeout, When the timer fires, Then it auto-submits. *Assert: state=submitted after the configured idle window (default provisional — value TBD pinned by ENG-2094).* [Source: mOzNDhn322 FR-STG2 §12 assumption]
- [ ] **AC3 (negative)** — Given an illegal transition (e.g. `applied→open`), When attempted, Then it is rejected. *Assert: CAS `UPDATE … WHERE state=$expected` affects 0 rows; error returned.* [Source: GOacy9kdJz AD-2]
- [ ] **AC4 (concurrency)** — Given two concurrent advances, When both run, Then exactly one wins. *Assert: one succeeds, one gets a CAS-loss error; no double outbox event.* [Source: GOacy9kdJz AD-2]
