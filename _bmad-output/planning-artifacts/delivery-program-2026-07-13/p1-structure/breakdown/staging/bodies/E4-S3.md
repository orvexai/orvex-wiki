## 🎯 Story

As a **knowledge manager** I want **a Review Queue API with deterministic Disposition buckets (auto-apply-eligible / recommended-accept / flagged / recommended-reject), filtered bulk accept/reject/merge, per-item drill-down diffs, and a mandatory per-item adjudication lane for below-confidence routes** so that **I can clear a 100-Proposal ChangeSet in ≤10 interactions while a confident mis-route can never hide inside a bulk skim**.

**Definition of Done:** one named test `TestReviewQueueBucketsAndAdjudicationLaneExcludesBulk` (integration — Dispositions roll up into the four deterministic buckets; below-confidence routes always land in *flagged* as a per-item lane excluded from bulk-accept regardless of Disposition; a common-case 100-Proposal set is clearable in ≤10 interactions). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2094); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given triaged Dispositions, When the queue renders, Then buckets roll up deterministically: auto-apply-eligible = apply-as-routed/apply-with-merge on low-risk ops from above-tier producers; recommended-accept = same otherwise; flagged = duplicate-of/conflict/escalate/divert-to-workgraph + all destructive; recommended-reject = reject. *Assert: bucket membership matches the rule exactly.* [Source: mOzNDhn322 FR-STG12]
- [ ] **AC2** — Given a below-confidence route, When bucketed, Then it joins *flagged* as a mandatory per-item adjudication (plain-English one-question) and is excluded from bulk-accept. *Assert: bulk-accept skips it even if its Disposition is apply-as-routed.* [Source: mOzNDhn322 FR-STG12; GOacy9kdJz AD-7]
- [ ] **AC3** — Given a 100-Proposal ChangeSet, When worked in the common case, Then it clears in ≤10 interactions. *Assert: interaction count ≤10 in the harness scenario.* [Source: mOzNDhn322 FR-STG12, NFR-STG1]
- [ ] **AC4 (error-path)** — Given a bulk accept spanning a flagged item, When submitted, Then the flagged item is rejected from the batch, not silently applied. *Assert: batch result excludes flagged with a reason.* [Source: mOzNDhn322 FR-STG12]
