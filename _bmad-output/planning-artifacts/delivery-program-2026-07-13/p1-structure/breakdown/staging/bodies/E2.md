## ΏθΟψ Feature area

Staged Proposal Intake is **the only agent write path into customer knowledge** (PRD ┬π4.1). Agents get one safe verb έΑΦ *propose* έΑΦ and submission is cheap, append-only, and never blocks on review. Every agent-intended change lands as a Proposal (Operation + target + DfM/PM content + base-version stamp + provenance) grouped per session into a ChangeSet. This epic builds the intake half of the state-machine pipeline: the `intake`, `lifecycle`, `quota` domain packages plus the S3 pointer-state store.

Why it exists: direct writes at fleet scale wreck the wiki (~200 round-trips, ~200 history rows, ~700 jobs per 100-doc session έΑΦ PRD ┬π1). Intake inverts that bet: proposals are disposable rows, the wiki DB is byte-for-byte untouched, and admission control keeps a write burst from ever degrading the engine or other tenants.
