## 🎯 Story

**Audit: already implemented** (evidence in §4) — this story is VERIFY + harden: prove the behaviour via the named DoD test + adversarial review; do not rebuild.

As a **wiki author and a consumer of the engine (wiki-api, ai)** I want **the transclusion-impact primitive (given a source page, resolve every page that transcludes it) plus the safeguard interceptor that guards destructive mutations of a transcluded source** so that **editing or deleting a page that other pages embed cannot silently break its dependents — the write is intercepted, the impact set is returned, and an unconfirmed destructive change is refused** (FR-10).

**Definition of Done:** ONE named test `TestTransclusionImpactAndSafeguardInterceptor` (integration layer — asserts through the primitive that a source transcluded by N pages returns those N in its impact set, and that a delete/breaking-edit of that source is intercepted (refused without confirm / surfaced with its impact) while a non-transcluded source is not intercepted). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2103); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a source page transcluded by other pages, When transclusion-impact is queried, Then it returns the exact set of dependent pages (via the `page-transclusions` relation). *Assert: impact set == the transcluding pages; scoped to the caller's tenant.* [Source: FR-10]
- [ ] **AC2** — Given a destructive mutation (delete / breaking content change) of a transcluded source, When the write is attempted, Then the safeguard interceptor intercepts it and refuses / requires confirmation, surfacing the impact set. *Assert: unconfirmed destructive change is refused; impact returned.* [Source: FR-10]
- [ ] **AC3** — Given a source page that nothing transcludes, When it is edited or deleted, Then the interceptor does not obstruct it. *Assert: no false-positive interception (impact set empty → pass-through).* [Source: FR-10]
- [ ] **AC4** — Given a cross-tenant transclusion probe, When impact is resolved, Then no cross-tenant dependent page is ever returned. *Assert: cross-tenant bytes = 0.* [Source: FR-10, G2]
- [ ] **AC5 (negative)** — Given transclusion *rendering / embed-resolution* (materialising the embedded body into output), When invoked, Then it is composition and lives outside the engine primitive — the engine exposes only the impact query + the interceptor. *Assert: no embed-render grammar in the engine primitive.* [Source: FR-10, Non-goals]
