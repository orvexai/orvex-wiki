## 🎯 Story

**Audit: already implemented** (evidence in §4) — this story is VERIFY + harden: prove the behaviour via the named DoD test + adversarial review; do not rebuild.

As a **standalone/embedded deployer** I want **the engine raw API kept internal-only with NO `/docs`/Swagger (wiki-api fronts all programmatic access, the React client the sole exception), every platform dependency reached via a configured URL, and core read/write degrading cleanly when knowledge/ai are absent** so that **a standalone or embedded-elsewhere wiki is a config change not a fork, and a platform outage never breaks page editing** (FR-W25, FR-W16, FR-W17, A-PORTABLE, A-APIBOUNDARY).

**Definition of Done:** ONE named test `TestEngineHasNoSwaggerAndDegradesGracefully` (integration layer — asserts through the engine that no `/docs` route exists, the client boots against a swapped set of `*_URL` endpoints, and core get/save/edit/list survive a simulated knowledge/ai outage). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2103); this story is dispatch-blocked until that tag exists.*
