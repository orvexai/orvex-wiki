## 🗺️ Feature area

The engine runs its own React client (renders ProseMirror natively on the engine API — **not** behind wiki-api) and is deliberately portable. This epic delivers the **collab + client** primitive (FR-W9 — Yjs/Hocuspocus with server-schema node/mark registration that must include every additive node or Yjs strips it, **minus all Linear NodeViews** which are never folded in, D-S11), the **thin-UI AI/MCP/API/api-key affordances** (FR-W18c / D-S4 named exception — bubble-menu quick-actions, Cmd+K/Cmd+J palette, `/ai` slash, Ask-search box, AI-diagram affordances, MCP + API settings pages, all as pure UI whose SSE handlers are `ReadableStream` readers with **zero server AI logic**), and the **API boundary + portability** (FR-W25 — engine raw API is internal-only, **no `/docs`/Swagger**, wiki-api fronts all programmatic access; FR-W16/FR-W17 + A-PORTABLE — every platform dependency is a configured URL, `KNOWLEDGE_URL`/`AI_URL`/`IDENTITY_URL`/`BILLING_URL`/`MCP_URL`/`CONSOLE_URL`, with graceful degrade so a knowledge/ai outage never breaks page editing).

Why it exists: G4 (product-family-agnostic — standalone/embedded is a config change, not a fork) and G5 (the client keeps working when the platform degrades).
