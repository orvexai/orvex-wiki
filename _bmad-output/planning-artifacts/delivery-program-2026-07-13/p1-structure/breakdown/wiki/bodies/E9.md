## 🗺️ Feature area

Two cross-cutting obligations that the engine — as the only AGPL artifact and the sole studio-spine producer — owes the family. First, the **cross-cell tenant-move typed contract** (A-MOVE / cell rule #10 — the non-retrofittable part): A-TENANCY covers only the in-place personal→Teams re-key, but moving a tenant *across cells* is a separate typed contract. The engine owns tenant-local state in four stores (Postgres rows incl. `orvex_page_meta`; S3 attachment prefixes; Redis quota counters; the outbox cursor), each of which must appear in a versioned `TenantMoveManifest{schema_version, stores, s3_prefixes, cursors}` plus a `TENANT_MOVE.md` **coverage-checked against the engine's declared datastores**; the engine ships the typed **quiesce/export/import/activate** step-API with mandatory `Idempotency-Key` (a 501 export/import *body* is acceptable day 1, the typed *contract* is not). Second, **per-role observability + ADR filing** (A-OBSERVE / A-CELL / A-ADR): each of the three roles carries its own liveness signal (worker relay-liveness + lag heartbeat as probe *and* OTLP metric; collab Hocuspocus WS probe) so a dead component never reads green (finding F5), `/api/health` **echoes `CELL_ID` + `CLUSTER_NAME`** (rule #4, finding F10), and the load-bearing rulings (D-S13, D-S17, D-S3, the `402 QUOTA_EXCEEDED` contract, the A-QUOTA fail-open tradeoff, D-S27) are filed as `draft ADR-NNNN`.

Why it exists: A-MOVE (non-retrofittable typed contract), A-OBSERVE (a dead relay silently stops all emission), A-ADR (CS §9 — file the rulings).
