## 🎯 Story

**Audit: already implemented** (evidence in §4) — this story is VERIFY + harden: prove the behaviour via the named DoD test + adversarial review; do not rebuild.

As a **downstream consumer (knowledge, billing)** I want **one outbox row committed in the SAME DB transaction as each domain mutation, covering the collab `onStoreDocument` write path** so that **event emission is transactional and lossless, replacing today's best-effort `EventEmitter2 → Redis XADD` dual-write that swallows errors** (FR-W5, FR-17, A-EVENTS).

**Definition of Done:** ONE named test `TestOutboxRowInSameTxAsCollabWrite` (integration layer — asserts through the collab persistence path that a page mutation and its `orvex_outbox` row commit atomically: rolling back the mutation rolls back the outbox row, and a committed mutation always has its row). *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2103); this story is dispatch-blocked until that tag exists.*
