## 🗺️ Feature area

The engine's sole auth code is **consume-only** (A-AUTH): it consumes an identity-minted exchange token and mints its session cookie; identity owns verification/mint (RS256/JWKS). This epic delivers the three auth-spine primitives that gate every satellite and the AGPL source publication: the **clean-room AGPL api-key rebuild** under `orvex/api-key` with `jwt.strategy.ts:83` repointed off `ee/api-key` (FR-W7 — today's live path is EE-derived and `packages/orvex-api-key` is an empty placeholder; a copy is **not acceptable** — this is a high-licensing-risk launch gate), the **exchange-token session-mint consume** (FR-W6 — accept identity-minted RS256/JWKS tokens replacing the symmetric `APP_SECRET` `jwt.verify`, mint session via `createSessionAndToken`; **native email/password login removed fully, no break-glass**, D-S3), and the **AGPL §13 source-offer** (FR-W19 — one canonical `@Public` `{sha, sourceRepo}` endpoint + public git mirror + UI source link + `LICENSE-COMPLIANCE.md`, reconciling the fork's two conflicting endpoints and the `orvexai/docmost` vs `orvex-ai/orvex` inconsistency to one value).

Why it exists: FR-W6 acceptance is the enabler that unblocks wiki-api Phase-0 and knowledge; FR-W7 + FR-W19 are AGPL-cleanliness launch gates that must land before the source is published.
