## 🗺️ Feature area

These are the load-bearing primitives every external consumer (wiki-api, knowledge, ai, the CLI) reaches across the network. The epic delivers, in the rollout's mandated order, the **`orvex_page_meta` side table FIRST** (FR-W3 — the schema-enemy move that gets 18 product columns off upstream `pages` so upstream stays pristine and the FR-30 gate can forbid new columns), then the **single `apply-ops` write chokepoint** (FR-W1 — typed PM-JSON ops, CAS `ifVersion`→409, atomic-or-409, the only write path both REST and Yjs funnel through), the **FR-13 ACL evaluation primitive** (FR-W2 — accessible-space-IDs + `filterAccessiblePageIds`, with the `evalPage` page-level-restriction fix and the delta-pull-through-ACL requirement), and the **export primitive** (FR-W4 — per-page markdown FR-18 + embed-resolved `text_repr` FR-38, scoped by the REM-1 IDOR authz fix and bound to a real audit service).

Why it exists: G2 (airtight multi-tenant safety — every read narrows through ACL, every write goes through the chokepoint) and the fact that knowledge/ai/wiki-api are *unbuildable* until these primitives exist and are stable.
