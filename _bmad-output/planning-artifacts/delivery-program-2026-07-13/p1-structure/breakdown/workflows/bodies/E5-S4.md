## 🎯 Story

**As a** workflows platform engineer
**I want** `ProvisionWorkflow` writing every newly provisioned tenant's Free-plan entitlement record via billing's idempotent create-Free step-API as an AUTHORITATIVE step (unbounded retry)
**so that** no tenant is left un-gated, and a transient billing outage stalls (never fails) the provision while the engine's F-QUOTA fail-closes to Free defaults — so a stalled create never over-grants.

**Definition of Done:** one named test `TestFreeEntitlementCreateAuthoritative` — workflow-test-env layer: provision calls billing create-Free AUTHORITATIVELY; a billing outage stalls-and-retries (workflow stays Running); the call is idempotent under replay

Note: *final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2107); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given any provisioned tenant (user-keyed or org), When ProvisionWorkflow runs, Then it calls billing create-Free with Free-tier defaults — *create-Free called once per tenant; billing owns VALUES* [Source: FR-W18 / D-S17]
- [ ] **AC2** — Given a billing outage, When create-Free is attempted, Then the AUTHORITATIVE step stalls-and-retries (workflow does not fail) — *workflow Running through outage; MaximumAttempts==0* [Source: FR-W18 / NFR-W5]
- [ ] **AC3** — Given a redelivered provision, When create-Free is called again, Then the entitlement is created once (negative: no double-grant) — *idempotency key → one entitlement record* [Source: FR-W18 / NFR-W2]
- [ ] **AC4** — Given this service, When entitlement values are considered, Then it never computes them (negative) — *no entitlement VALUE computed here; only the create call* [Source: FR-W18 / A-W8]
