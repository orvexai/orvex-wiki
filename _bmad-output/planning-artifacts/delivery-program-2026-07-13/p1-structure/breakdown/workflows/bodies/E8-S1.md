## 🎯 Story

**As a** workflows platform engineer
**I want** the `OrgUpgrade` workflow (D-S17 upgrade pass): mint the Clerk org at upgrade, re-key the user-keyed tenant to the org in place (`user:<id>`→`org:<id>`), carry the tenant's data + entitlements, start seat counting — orchestrated across identity (org mint + registry re-key + FR-I8 access transition), engine (workspace re-mapping), and billing (entitlement re-key) — durable, resumable, and compensating, dedup `org-upgrade-<tenant>`
**so that** a personal user upgrades to Teams without a half-converted principal: any authoritative-step failure halts resumably and rolls back to the pre-upgrade user-keyed tenant, and new members never retroactively see pre-conversion private content.

**Definition of Done:** one named test `TestOrgUpgradeCompensatesOnFailure` — workflow-test-env layer: OrgUpgrade re-keys the tenant carrying data+entitlements; injecting an authoritative-step failure triggers compensation back to the pre-upgrade user-keyed tenant (no half-converted state)
