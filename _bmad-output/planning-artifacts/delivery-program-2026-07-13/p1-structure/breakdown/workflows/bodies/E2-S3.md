## 🎯 Story

**As a** workflows platform engineer
**I want** workflow input/result JSON types versioned append-only so in-flight workflows decode across deploys
**so that** a deploy during a 30-day grace sleep never breaks decoding of an in-flight workflow's input/result.

**Definition of Done:** one named test `TestWorkflowTypesDecodeAcrossVersions` — unit/contract layer: an old-version input/result fixture decodes cleanly against the current type (append-only, no field removed/retyped)

Note: *final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2107); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a prior-version workflow input fixture, When decoded by current code, Then it decodes without error — *old-shape fixture unmarshals; new fields default* [Source: FR-W16 / NFR-W1]
- [ ] **AC2** — Given a field-removal or retype PR, When CI runs, Then the append-only check fails (negative) — *schema-diff lint exit != 0 on a breaking change* [Source: FR-W16]
- [ ] **AC3** — Given GraceHours in the input, When a policy value changes, Then it rides the input not an in-workflow env read — *no `os.Getenv`/`time.Now` inside workflow code; policy from input* [Source: NFR-W1 / A-W5]
