## 🎯 Story

**As a** workflows platform engineer
**I want** durable tool-loop workflows for orvex-studio-ai: loop state (conversation/step ledger, tool budget, iteration count) in the workflow; every model call and every tool execution as one activity calling an ai step-API; hard caps (max iterations, max wall-clock, token budget) on the input; `ContinueAsNew` bounding history on long loops
**so that** a crash mid-loop resumes instead of restarting, and non-deterministic LLM output enters history only as recorded activity results.

**Definition of Done:** one named test `TestToolLoopResumesAndCaps` — workflow-test-env layer: a tool loop resumes from its step ledger after a worker restart, enforces the input caps, and records model output only as activity results

Note: *final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2107); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given a tool loop, When a worker crashes mid-loop, Then it resumes from the step ledger (not restart) — *resume from last recorded step; no re-execution of completed steps* [Source: FR-W11]
- [ ] **AC2** — Given the input caps, When the loop hits max iterations/wall-clock/token budget, Then it stops at the cap (negative) — *loop halts at each cap; no overrun* [Source: FR-W11]
- [ ] **AC3** — Given non-deterministic LLM output, When it enters the loop, Then it is recorded only as an activity result — *no model call inside workflow code; only via activity* [Source: FR-W11 / NFR-W1]
- [ ] **AC4** — Given a long loop, When history grows, Then ContinueAsNew bounds it — *ContinueAsNew invoked on long loops* [Source: FR-W11]
