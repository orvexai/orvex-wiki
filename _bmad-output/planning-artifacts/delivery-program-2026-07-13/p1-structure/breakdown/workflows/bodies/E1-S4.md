## 🎯 Story

**As a** workflows platform engineer
**I want** a `WorkflowReplayer` CI job that replays recorded histories of each workflow type against current code, plus `workflow.GetVersion` gating every workflow-logic change
**so that** the #1 operational risk — a deploy replaying a live 720-hour grace history non-deterministically — has an automated guard.

**Definition of Done:** one named test `TestReplayRecordedHistories` — replay layer: `worker.NewWorkflowReplayer().ReplayWorkflowHistory` over committed histories for Provision/Deprovision/Reconcile asserts no non-determinism error

Note: *final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2107); this story is dispatch-blocked until that tag exists.*

## ✅ Acceptance Criteria

- [ ] **AC1** — Given recorded histories for each workflow type, When the replayer runs current code, Then no non-determinism error is raised — *`ReplayWorkflowHistory` returns nil for every recorded history fixture* [Source: NFR-W1 / A-W5]
- [ ] **AC2** — Given a workflow-logic change without a `workflow.GetVersion` guard, When CI runs, Then the replay of an in-flight-shaped history fails (negative) — *replay job exit != 0 on the ungated change fixture* [Source: A-W5]
- [ ] **AC3** — Given a 720h grace-sleep history mid-flight, When replayed post-change, Then the timer + purge decision replay clean — *the grace-sleep history replays with the purge branch intact* [Source: NFR-W1 / Arch §5]
