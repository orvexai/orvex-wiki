### 🎯 Story
As a **security-conscious tenant** I want **one `authz` chokepoint enforcing namespace isolation, the 3-state privacy enum, and per-item grants on every read and mutation** so that **no surface bypasses authorization and no grant ever crosses the namespace boundary**.

**Definition of Done:** one named test `TestGrantsWidenNeverCrossNamespace` (authz domain layer) asserts the privacy enum is the visibility floor, explicit grants may only widen it (never revoked by a stricter enum), and the namespace boundary caps both — evaluated identically across MCP, CLI, interop, step-APIs, and `ranking`. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2095); this story is dispatch-blocked until that tag exists.*

### ✅ Acceptance Criteria
- [ ] **AC1** — Given namespace-prefix isolation (`tenant / agent / session`), When any access is evaluated, Then no grant can cross it. *assert: a grant naming a cross-namespace principal is ineffective.* [Source: FR-MEM23, AD-9, NFR-MEM4]
- [ ] **AC2** — Given the 3-state privacy enum (`open|private|shared-private`) inherited from the user-memory design, When evaluating visibility, Then it is the default floor and per-item agent/role grants may only WIDEN it. *assert: `private` + a read grant → visible to grantee only; enum never revokes a grant.* [Source: FR-MEM21, FR-MEM23, AD-9]
- [ ] **AC3** — Given every surface (MCP, CLI, interop adapters, step-APIs, `ranking`), When it accesses data, Then it passes through the single `authz` package. *assert: import-lint proves no surface evaluates grants itself.* [Source: AD-9]
- [ ] **AC4** — Given the `system` principal (janitor, reaper, relay), When it acts, Then it carries a scoped capability set (consolidate/compact facts + closed items, revert leases), audited distinctly, and can NEVER touch open/in-progress/claimed items or cross a tenant. *assert: system principal denied on live items + cross-tenant.* [Source: AD-9]
- [ ] **AC5 (negative)** — Given a cross-tenant read attempt, When evaluated, Then it returns zero bytes (the isolation probe). *assert: cross-tenant bytes=0 probe green.* [Source: NFR-MEM4, AD-3]
- [ ] **AC6 (error-path)** — Given a Remembered-Fact key collision, When written, Then uniqueness on `(tenant, scope, key)` is enforced and recall resolves `session → agent → tenant`. *assert: unique key + resolution order.* [Source: FR-MEM8, AD-9]
