### 🎯 Story
As a **chat-agent whose workgraph is momentarily unavailable** I want **prime/recall to degrade to honest-empty-with-status while claim stays a server-side CAS** so that **my platform is never blocked and I never get fabricated context or a double-claim from a stale read**.

**Definition of Done:** one named test `TestDegradedModeHonestEmptyClaimCasSafe` (server/domain layer) asserts that under store degradation prime/recall return an honest-empty payload with a `{kind: degraded}` banner (never fabricated), and a coordination write (`claim`) still executes as a server-side atomic CAS that fails on conflict. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2095); this story is dispatch-blocked until that tag exists.*

### ✅ Acceptance Criteria
- [ ] **AC1** — Given the workgraph store is unavailable, When `prime`/`recall` run, Then they return honest-empty-with-status, never fabricated content, never a blocked agent. *assert: degraded → empty payload + `{kind: degraded, scope, detail}` banner.* [Source: NFR-MEM6, AD-13]
- [ ] **AC2** — Given degraded mode, When a `claim` arrives, Then it still executes as a server-side atomic CAS — a stale-blind claim fails on conflict rather than double-claiming. *assert: reads degrade, writes do not get blind trust; CAS-fail on conflict.* [Source: NFR-MEM6, AD-2]
- [ ] **AC3** — Given knowledge is unavailable (not the store), When search runs, Then it is a `staleness` banner over the Postgres legs, never an empty response, never a `degraded` banner. *assert: knowledge-down → `{kind: staleness}` + Postgres results.* [Source: AD-13, AD-3]
- [ ] **AC4 (negative)** — Given any degradation, When the agent is guided, Then guidance is read-only proceed — coordination writes wait but are never fabricated as successful. *assert: no fabricated success; read-only-proceed guidance surfaced.* [Source: NFR-MEM6]
- [ ] **AC5 (error-path)** — Given an error is surfaced to an agent, When rendered, Then it never contains a copy-pasteable destructive command (beads ADR-0002 "the text was the bug"). *assert: no destructive command in any degradation message.* [Source: Architecture Agent-safety]
