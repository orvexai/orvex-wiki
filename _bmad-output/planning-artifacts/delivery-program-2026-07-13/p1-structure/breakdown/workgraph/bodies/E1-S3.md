### 🎯 Story
As a **workgraph build agent** I want **auth wired exclusively through `orvex-studio-lib/pkg/auth` with deny-by-default and per-agent scoped tokens** so that **no verb executes for an unverified principal and no service ever calls an IdP directly**.

**Definition of Done:** one named test `TestDenyByDefaultAuth` (server/integration layer) asserts that a request with no/invalid token is rejected before any domain call, and a valid per-agent token yields a principal carrying `tenant/agent` scope. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2095); this story is dispatch-blocked until that tag exists.*

### ✅ Acceptance Criteria
- [ ] **AC1** — Given a request with no token, When it hits any verb, Then it is denied before domain logic runs. *assert: 401/deny; zero domain-package calls (spy on the seam).* [Source: P4/ADR-0009]
- [ ] **AC2** — Given a valid per-agent token, When verified, Then the principal carries `{tenant, agent, roles}` scope. *assert: extracted claims match the token.* [Source: NFR-MEM4]
- [ ] **AC3** — Given a high-privilege verb (grant mutation, admin pin), When invoked, Then `VerifyFresh` is required. *assert: a stale-but-valid token is rejected on `VerifyFresh` paths.* [Source: P4/ADR-0009]
- [ ] **AC4 (negative)** — Given a token from an unregistered issuer, When verified, Then it is denied. *assert: unknown-issuer token → deny.* [Source: P4]
- [ ] **AC5 (error-path)** — Given the verifier cannot reach JWKS, When a token arrives, Then it fails closed (deny), never fails open. *assert: JWKS-unavailable → deny, not allow.* [Source: no-fallbacks, deny-by-default]
