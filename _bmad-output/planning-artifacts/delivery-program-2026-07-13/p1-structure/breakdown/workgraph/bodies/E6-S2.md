### 🎯 Story
As an **org admin** I want **workspaces under one org principal to share a workgraph namespace via explicit admin grant** so that **intra-org agents coordinate on shared work while distinct tenants stay fully isolated**.

**Definition of Done:** one named test `TestOrgSharedNamespaceIntraOrgOnly` (authz domain layer) asserts a polymorphic `{user|org}` org principal can grant intra-org namespace sharing, while arbitrary cross-tenant sharing between distinct tenants remains impossible. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2095); this story is dispatch-blocked until that tag exists.*

### ✅ Acceptance Criteria
- [ ] **AC1** — Given workspaces under one org principal, When an admin grants sharing, Then they share a workgraph namespace (polymorphic `{user|org}` tenancy). *assert: intra-org workspaces see the shared namespace after the grant.* [Source: FR-MEM25, AD-9]
- [ ] **AC2** — Given two distinct tenants, When any sharing is attempted, Then it is impossible — the isolation boundary is unchanged. *assert: cross-tenant share request denied; bytes=0 probe holds.* [Source: FR-MEM25, PRD §5]
- [ ] **AC3** — Given a personal (org-free) user, When they upgrade to an org, Then the polymorphic tenancy carries data under the org principal without an org-only assumption breaking. *assert: personal-tenant probe passes; org-only assumption is a bug.* [Source: ADR-0001/0012, FR-MEM25]
- [ ] **AC4 (negative)** — Given the admin grant is revoked, When evaluated, Then the shared namespace access ends (no lingering cross-workspace read). *assert: post-revoke access denied.* [Source: AD-9]
