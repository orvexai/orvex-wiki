### 🎯 Story
As a **workgraph build agent** I want **a transactional outbox plus `cmd/relay` publishing `studio.workgraph.*` CloudEvents to `studio-spine`** so that **every mutation is audited on the spine with at-least-once delivery and no polling or Redis bridge**.

**Definition of Done:** one named test `TestOutboxAtLeastOnceDedup` (event/integration layer) asserts that a mutation writes its event in the same transaction, the relay publishes it, and a replayed duplicate dedups by its deterministic event id. *Final H1–H17 elaboration + exact contract tag/versions are pinned at pack certification (ENG-2095); this story is dispatch-blocked until that tag exists.*

### ✅ Acceptance Criteria
- [ ] **AC1** — Given any mutation, When committed, Then its outbox row is written in the same DB transaction (never a second connection). *assert: rollback of the mutation rolls back the event.* [Source: AD-7, P2]
- [ ] **AC2** — Given `cmd/relay`, When it drains the outbox, Then it publishes ADR-0007-enveloped events (required set + `orvexcell`/`orvextenant`, `partitionkey`=tenant) to `studio-spine`. *assert: published envelope matches the contract schema.* [Source: ADR-0007/0010, AD-7]
- [ ] **AC3** — Given a replayed event, When re-published, Then consumers dedup via the deterministic id derived from `(Idempotency-Key, resource_id)`. *assert: duplicate id → single logical apply.* [Source: AD-7]
- [ ] **AC4 (negative)** — Given the relay is down, When mutations continue, Then events accumulate durably in the outbox and drain on recovery (no loss, no in-band coupling). *assert: relay-restart drains the backlog fully.* [Source: NFR-MEM5]
- [ ] **AC5 (error-path)** — Given a malformed event, When the relay encounters it, Then it fails loudly and does not silently drop. *assert: poison event surfaces an error, not a swallow.* [Source: no-fallbacks]
