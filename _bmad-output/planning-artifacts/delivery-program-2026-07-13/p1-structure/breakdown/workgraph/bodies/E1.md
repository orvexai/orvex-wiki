## 🧭 Epic — Foundations: repo, contracts codegen, Postgres store, auth, event spine

Everything in the workgraph service stands on a thin, correct platform floor. This epic
builds the greenfield `orvex-studio-workgraph` repo in the six-tier family shape, wires it
to the pinned `orvex-studio-contracts` tag (all verb + event + step-API schemas codegen from
one tag — nothing hand-authored), stands up the sole Postgres driver with UUIDv7 keys,
fail-closed RLS, and the embedded-FS forward-only migration applier, imports the shared
`orvex-studio-lib/pkg/auth` verifier (deny-by-default, per-agent scoped tokens), and lights
the transactional outbox → `cmd/relay` → `studio-spine` seam that every mutation audits
through. It also mints the human-facing identity of a work item: server-issued per-tenant
short ids (`wg-` + base32-crockford), grammar pinned in contracts so every downstream parser
agrees.

Why it exists: AD-1 declares workgraph a greenfield service holding agent coordination state
only — it starts empty, migrates nothing, and every later epic (core, claims, facts, janitors,
authz, surfaces, console) compiles against this floor. Per the current-state map, `orvex-studio-lib`
is still a 52-byte deny-by-default scaffold and `orvex-studio-knowledge` is past a 501 scaffold —
so this epic's auth and event seams are the family's critical path made local.
