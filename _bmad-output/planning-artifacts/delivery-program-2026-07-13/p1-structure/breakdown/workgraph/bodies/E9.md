## 🧭 Epic — Rituals, adherence & wiki-worthy: the chat-platform hooks

Chat platforms have no SessionStart/PreCompact hooks — the rituals ride prompts, tool
descriptions, and server instructions. This epic makes that a first-class, versioned product
surface. It delivers the Prompt Pack store (ChatGPT-portal blocks, Claude project instructions,
CLI snippets) with admin-approved versioned revisions and per-tenant overrides, plus the
server-side reinforcement that ships with them (compact prime via MCP `instructions` at connect,
tool-description micro-prompts, stale-claim nudges in tool results). It delivers adherence
telemetry (prime-rate, claim-before-work, close-rate, stale-claim, discovered-from) derived from
tool-call events over the observable denominator (workgraph-active sessions), the pre-GA
adherence spike that sets the viability floor, and the GA gate. Finally it delivers the
wiki-worthy flow: marking an item/note wiki-worthy packages distilled content and submits a
Staging Proposal via `stagingclient` — never the wiki directly — with cross-links both ways and
loud cycle rejection on unchanged resubmission.

Why it exists: SM-3 (ritual adherence) and UJ-4 (working memory matures into wiki knowledge).
AD-11 makes packs versioned artifacts, forbids silent prompt drift, and pins the shared
feedback/adherence event envelope (authored once in contracts, shared with staging so the later
learning-loop merge is additive). AD-12 keeps wiki-worthy flowing out through staging only.
