## C1 — [REFUTE, serving-service] P0 `/v1/audit` row routes the write to an "engine WORM sink" — contradicts settled canon R25 (emit-only)

- **Target row:** TIER P0 — "`/v1/audit` external-agent audit-write (usable)". Serving-service column reads: **"wiki-api (emit) → engine WORM sink; audit owns read/query (R16/R25)."**
- **What is wrong:** The "→ engine WORM sink" leg is a **PO-D4 / R25 violation**, and the draft cites R16/R25 while contradicting them. `evidence-decisions.md §4` (R25, `po-decisions-2026-07-07.md:1462`) is explicit: *"NO service owns audit storage, WORM tables, audit read-models, or audit query APIs. Every service EMITS audit events (outbox, per its existing event discipline); the dedicated audit+compliance service OWNS storage… Do not design wiki-api/MCP/CLI to own audit storage, **a WORM table**, an audit read-model, or an audit query endpoint."* An "engine WORM sink" is precisely the storage sink R25 removes from every repo (the engine is SoR for **document writes+collab**, not for the audit WORM store). The draft copied the framing verbatim from ticket **ENG-2538** ("External-agent audit-write endpoint → engine WORM sink", `evidence-linear.md:605`) — the exact "the ticket had it" trap PO-D2 says to refute. Memory `audit-compliance-service-incoming` + R25 both say Daniel's incoming audit service supersedes this seam.
- **The fix:** Serving-service → **"wiki-api EMITS an audit CloudEvent (outbox, existing event discipline); `orvex-studio-audit` owns storage + retention + tamper-evidence + read/query (R16/R25, canonical PRD `vhC5XXCYkC`)."** Scope the wiki-api work as an **emitter reservation stub** (event name/shape provisional, contract internals NOT invented — R25 posture), not a write-to-engine-sink. Note in the row that ENG-2538's own wording is misaligned with R25 and must be re-scoped, not scheduled as-is.
- **Evidence:** `evidence-decisions.md §4` (R16/R25 verbatim + "Constraint on the redo"); `evidence-linear.md:605` (ENG-2538 wording); `evidence-fork-server.md §3.6` (the fork's in-process engine sink is the *old* model).
