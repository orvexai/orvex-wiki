## 2. Deeper summaries — load-bearing pages

### Architecture (`o2waDNw3ix`, canonical)
- No-service repo: only executable surface is self-validation CI; conformance/drift gates run in consumer repos' CI pinned to a release tag.
- A-LAYOUT fixes one directory per contract kind (openapi/, events/, sse/, sources/, identity/, billing/, errors/, fixtures/, gates/, mcp/, obs/, SEAMS.md).
- A-AUTHOR: one-way conformance — hand-authored here, engine conforms to it, no engine-derived artifact ever committed; PR provenance attestation + generator-signature lint as the guard; DfM fixtures likewise clean-room (from documented schema + engine's published block catalog, never from AGPL TS).
- A-CATALOG: CloudEvents 1.0, per-type profile (thin/rich), orderingkey, publisher registry identity; monotonic per-document `version` = persisted write-commit counter (never a relay-stamped sequence); batch fan-out to per-page CloudEvents (D-CON-4); catalog is forward-authored ahead of the unbuilt outbox→Kafka relay; `billing.entitlement.changed` added; no `linear.*` namespace (D-S11).
- A-SSE: EVENTS.md = verbatim catalog import, WIRE.md = extracted behavior (replay-then-tail, Last-Event-ID resume, `/events/head` bootstrap, monotonic+epoch cursor v1/v2, legacy unprefixed frame names, thin `content_updated{pageIds[]}`, `permission.changed` payload w/ `added`→full resync, 1 MiB/512-queue, <45s heartbeat+backoff constants, `applied_events` dedup, 7-day minimum resume window as a wire term). Frame visibility = key-type filter ∩ token scope (D-CON-7).
- A-OPENAPI: one file per seam server incl. wiki-api (`openapi/wiki-api.yaml`, FR-C19 — verb grammar `search/get/save/edit/list`, no `ask`) and ai (`openapi/ai.yaml`, FR-C22 — cited-ask/RAG K5 schema, spend-cap read/enforce, cap value owned by billing); bearer-tier vs workload-identity-tier (mTLS/SPIFFE) split is absolute — internal/ servers reject bearer outright.
- A-DFM: one authored PM-JSON↔DfM parity corpus (21 embeds post-Linear-removal), two independent clean-room serializers (Go `pkg/dfm` in lib CI, engine's AGPL TS in engine CI) that must never read each other; corpus lives in contracts (Apache side) not lib, per D-CON-8.
- A-GATES: gate definitions + reference tooling live here, execution happens in consumer CI, always pinned to a release tag; FR-30 divergence gate ships report-only (red-day-one reality), AGPL-import guard enforces from day one.
- A-VERSION/A-CODEGEN: repo-level semver, immutable tags, additive-by-default; consumers codegen at build from a pinned tag, no generated code committed here — shared Go output lives in orvex-studio-lib.
- Tightened revision (T1-T7) resolutions: **T1** ADR-0008 settles change-authority as two layers — artifact edits are automated-CI-only (no human ratify), the DESIGN DECISION behind a CS §9-triggering change needs an ADR + human doc-ratify; classifier = FR-C17 drift-gate/semver policy, ambiguity fails safe to the ADR lane. **T2** wiki-api host topology: adopt flat public host (`wiki-api.orvex.dev`) with per-cell `origin.{cell}.orvex.dev` marked internal/x-origin — routing mechanism (Worker vs Traefik) is OPEN DECISION #2. **T3** `ai.yaml` is the single home for K5/ask; wiki-api's Phase-0 proxy paths are `x-deprecated`+`x-superseded-by: ai.yaml`, retired at cutover — timing (immediate vs coexist window) is OPEN DECISION #3. **T4** event `version` MUST be a required field on every content-bearing schema; pinned schemas MUST set `additionalProperties: false`. **T5** four v0.1 keystones sequenced as blocking: validate.yaml CI, SEAMS.md, agpl-import guard, VERSION+CHANGELOG+first tag. **T6/T7** gate/schema hygiene + authority/honesty nits (all since resolved per later pages — see Staleness).

## 3. Bindings on the AI-surfaces redo (MCP / wiki-api / CLI)

These are commitments in this space that constrain the PRD/architecture redo for the three AI-facing surfaces:

- **Verb grammar is fixed and narrow.** `openapi/wiki-api.yaml` (FR-C19, still `draft` x-status, not yet GA-pinned) owns exactly `search/get/save/edit/list` over `{resource_type, locator}`. **`ask` is explicitly NOT a wiki-api verb** — D-ASK assigns the entire cited-ask/RAG loop to `openapi/ai.yaml` (FR-C22) and orvex-studio-ai as owner. Any MCP/CLI redo that routes "ask"-shaped intent through wiki-api directly contradicts a binding, adjudicated (T3) decision.
- **wiki-api's current shipped `ask`/`related`/`search` paths are a Phase-0 transparent proxy only** (`x-proxy: engine`), marked `x-deprecated: true` / `x-superseded-by: ai.yaml`, to be retired at the ai.yaml cutover — timing (immediate vs short coexist window) is still OPEN DECISION #3. The MCP/API/CLI redo should not build durable dependencies on the wiki-api ask proxy.
- **Write receipt shape is pinned**: `{url, id, version, persisted}` for page and block writes; block writes additionally carry `changedBlockIds` + applied/skipped disposition; `version` MUST be populated (closes a gap where the shipped `postBlock` returned only `{block_id, status}`).
- **CAS/optimistic-concurrency representation is pinned**: `ifVersion` is a monotonic version/etag, NEVER a wall-clock `updated_at` — 409 `VERSION_MISMATCH` compares like with like. Load-bearing for any CLI/MCP write-path design.
- **QUOTA_EXCEEDED (402) propagates unchanged** through wiki-api — never cached, never partial-written-around; batch atomicity means the whole patch fails; carries the upgrade/entitlement deep-link. Any MCP/CLI tool surface touching writes must surface this uncaught.
- **Auth is a strict two-scheme split**: bearer (referencing principal/token-scope schemas, C5 space-scoped semantics — a scoped token can never read outside its spaces) for public/engine/wiki-api/ai surfaces; workload-identity (mTLS/SPIFFE, `x-rejects-bearer: true`) for `x-tier: internal` surfaces and the identity lifecycle mirror. A codegen'd client must never present bearer creds to an internal-tier server.
- **Error/exit-code vocabulary is frozen and family-wide**: ~58 errorCodes (`errors/vocabulary.yaml`) + CLI 0-9 exit-code buckets (`errors/exit-codes.yaml`), FR-C23 — the CLI transports these verbatim; namespaced extension codes (`AI_*`/`SEARCH_*`) may be added without renumbering exit buckets. `LINEAR_NOT_CONNECTED` is explicitly removed, not reserved.
- **SSE wire contract (FR-C10) is the CLI's freshness mechanism** and is extremely detailed and load-bearing: `/events/head` cold-start bootstrap, monotonic+epoch-ms cursor (v1 legacy Redis IDs / v2 `<topic-epoch>:<offset>`), legacy UNPREFIXED frame names on the wire (not `wiki.*` — `PROFILES.md`'s type↔frame map is authority), thin `content_updated{pageIds[]}` with no body (client must refetch raw PM), `permission.changed{affectedUserId, changeType}` where `added` forces a whole-workspace resync, 1 MiB max line + 512-frame queue, <45s heartbeat + specific backoff constants per status code, `applied_events` exactly-once dedup, and a pinned **7-day minimum client-reliable resume window**. Effective frame visibility = key-type filter ∩ token scope (D-CON-7) — a scoped token must never see frames outside its spaces regardless of key class. Any MCP-streaming design (R21: streaming folded into existing verbs, per memory) must reconcile against this wire contract if/when it rides the same SSE gateway, or explicitly state it does not.
- **Codegen discipline (ADR-0035, FR-C25/26) governs how MCP/CLI/api-server generate typed clients**: source is ALWAYS a local checkout of the contracts repo (`ORVEX_STUDIO_CONTRACTS_DIR`), NEVER the AGPL engine's served descriptor — generating from the served descriptor is explicitly FORBIDDEN (clean-room boundary). Draft artifacts MAY be generated against for in-flight dev; a GA cutover's recorded pin MUST reference an `x-status: pinned` artifact at a released tag. Each consumer records `contractsRelease` (tag) + `orvexContractsPin` (per-artifact SHA) in its own package.json/manifest; regen is a deliberate reviewed PR with a determinism gate, never a background refresh. mcp is the ONE seam this is already proven on end-to-end (committed `scripts/gen-client.ts`, checked-in `generated.d.ts`, live determinism gate) — the api and ui codegen paths are NOT yet built. The MCP/API/CLI redo should generalize the mcp pattern rather than reinvent it, and must not silently pin unreleased (`studio.staging.*`/`studio.workgraph.*`) event types that exist only at HEAD, not at any tag.
- **AGPL-import guard (FR-C21) is a hard, always-on CI gate** in all 14 registered closed consumer repos including presumably mcp/api/cli's home repos: zero `@docmost/*` imports, zero copied code, enforced from day one (never an allow-list entry, always a licensing incident to remove).
- **Change-authority discipline (ADR-0008) applies to every contracts edit the redo requires**: additive/optional changes ride the automated CI-only lane; any breaking change (removing/renaming a field, changing a required set, altering error/exit-code semantics, retiring an endpoint, or forcing a coordinated multi-repo cutover) — e.g. finally retiring the wiki-api ask proxy, or promoting `ai.yaml`/`wiki-api.yaml` rows from draft to pinned in a way that changes shape — requires an ADR + human doc-ratify + a design-it-twice note + a documented dual-publish/deprecation window. **The human ADR-lane ratifier is still unnamed (MR-1/OQ-C1)** — any redo work landing in the ADR lane is currently BLOCKED pending that naming, not decidable by an autonomous agent.
- **The AI spend-cap / entitlement shape is contested and unresolved (MR-2/MR-3/MR-4)** — any AI-surface work (ai.yaml pinning, MCP AI tools, cap-aware CLI UX) that assumes a specific cap field name, currency, or Free-tier entitlement model (lifetime counter vs allowlist) is building on ground that two canonical/shipped sources directly contradict. This is a program-level blocker for the ai.yaml GA cutover specifically, not just a contracts housekeeping item.
- **wiki-api's public host is flat** (`wiki-api.orvex.dev` dev / `.orvex.ai` prod) with per-cell origins internal-only and 421-based re-resolve at the origin — any MCP/CLI client design should target the flat host, not a per-cell URL, per the T2-resolved architecture.
- **F-QUOTA / quota-status is a read primitive on engine-primitives**, not owned by wiki-api or ai — MCP/CLI tooling that surfaces quota state should read through the appropriate primitive rather than re-deriving quota logic.
