## 2. Deeper summaries ‚Äî load-bearing pages

### Architecture (`0NPvDR3C0m`, canonical) + Audit (`JxnqaaYx3j`, canonical)

- 10-package topology, each package a hard boundary; no package imports a satellite; `pkg/auth`/`pkg/events`/`pkg/obs`/`pkg/dfm` import no other lib package.
- `pkg/auth` = the family security ceiling: dual-IdP (Clerk + Keycloak) multi-issuer JWKS verify ‚Üí deny-list ‚Üí bounded cached introspection ‚Üí deny-by-default scope gate on **effective scope = token-claim scope ‚à© latest introspection**; identity itself imports this same verifier (single implementation, no spec-vs-impl drift). 12 numbered conformance guarantees (V1‚ÄìV12) in `pkg/auth/authtest`, run 3 ways (lib CI, identity CI, every satellite CI).
- **Cell-tenancy apparatus (A-CELL) is folded into existing packages, not a new one**: `pkg/auth` freezes `cell`/`cell_epoch` claim wire-names + mounts cell-guard middleware (`principal.cell != CELL_ID` ‚Üí 421, distinct from the 403 scope-gate); `pkg/events` stamps the `orvexcell` CloudEvents extension attr and fails closed on consume-side mismatch; `pkg/obs`'s `/healthz` echoes `CELL_ID`+`CLUSTER_NAME`; `pkg/kafkautil` names topics `{domain}-events.{cell}` (single-partition, cell-opaque). This is stated as **day-1, not retrofittable** (cell contract adopted while every service was a 2-file stub).
- Event path is explicitly the engine's **transactional outbox ‚Üí relay ‚Üí Kafka** (canon Principle 2) ‚Äî **no Redis‚ÜíKafka bridge, no Redis Streams stage**; lib carries zero bridge/relay plumbing (that's the engine's TS concern).
- `pkg/dfm` = clean-room (never AGPL-derived) Go DfM‚ÜîProseMirror serializer, 4-function surface (`PMToDfM`/`DfMToPM`/`MarkdownToPM`/`MarkdownToDfM`), invariant `DfMToPM(PMToDfM(doc)) ‚â° doc`; shared by orvex-wiki-api **and the Orvex CLI** (in-process, no engine `/v1/convert` calls); gated by a parity CI test against contracts golden fixtures + the engine's block-schema catalog; oracle for PM‚ÜíDfM is the engine's shipped DfM read-path renderer, not the legacy turndown export path.
- `pkg/engineclient`: F-QUOTA `QUOTA_EXCEEDED` (402) typed as a **terminal, non-retryable** error, propagated verbatim through wiki-api/mcp/ai.
- `pkg/billingclient`: reads billing's plans/entitlements SoR **fail-closed** ‚Äî unreachable ‚áí deny, never allow.
- Audit verdict: `contradicts-canon` **solely** on the cell dimension pre-tightening (now fixed-in-draft); otherwise exemplary (auth ceiling, fail-closed typing, AGPL clean-room, codegen-pin discipline all praised). Open decisions (OQ-L1..L7) remain genuinely open, not defects. ADR registry doesn't exist yet (TBD at Studio Act-1) so several decisions (A-VERIFY-IMPL, A-DEPS, A-DFM, A-CELL, A-VERSION, OQ-L5-closure) are recorded inline pending filing as discrete ADRs.

## 3. Bindings on the AI-surfaces redo (MCP / wiki-api / CLI PRD-architecture)

These are commitments in orvexstudiolib that constrain or must be reused by the redo of orvex-wiki-api, orvex-studio-mcp, and orvex-cli:

- **Auth is a pinned dependency, not something the redo designs.** All three AI-facing surfaces MUST verify callers via `pkg/auth`'s dual-IdP verifier + `authtest` V1‚ÄìV12 conformance suite (already shipped, mature, 471-line real implementation ‚Äî do not treat as a blocker or re-derive verification logic). The redo's PRD/architecture should cite lib's cell-guard (421) and scope-gate (403) as distinct steps a route sits behind, and should require each surface's CI to import + pass `authtest` V1‚ÄìV12 (this is currently UNVERIFIED for any consumer per the SDD ‚Äî the redo could be the first pack to close that gap).
- **Cell-tenancy is mandatory and day-1**: MCP/wiki-api/CLI, once deployed as satellites, must carry the `cell`/`cell_epoch` claim handling and the 421 cell-guard via `pkg/auth`, and any event they emit/consume must carry/verify the `orvexcell` extension attribute via `pkg/events`. This is non-negotiable/non-retrofittable per cell contract `JGAUQRsw2g`.
- **F-QUOTA / quota-exceeded propagation is a fixed contract**: `pkg/engineclient` types the engine's 402 `QUOTA_EXCEEDED` as terminal/non-retryable; wiki-api, mcp, and ai (and by extension any redo) MUST propagate it unchanged (with entitlement/upgrade deep-link), never retry, never soften to partial success.
- **Billing entitlements are fail-closed, read-only via `pkg/billingclient`**: any surface gating a capability on plan/entitlement (e.g., MCP tool availability, CLI feature gating) must deny on billing-unreachable, never silently allow.
- **DfM serialization is a shared, in-process, clean-room dependency (`pkg/dfm`)** for both orvex-wiki-api and the Orvex CLI ‚Äî neither should call an engine `/v1/convert` endpoint (those can retire), and neither should hand-roll or derive its serializer from AGPL TS. The parity gate's golden corpus is currently inadequate (6 core-GFM pairs vs a needed 21-embed+opaque+table-cell+mention corpus) ‚Äî this is a stated prerequisite gap the redo's wiki-api phase-1 exit should not assume is closed.
- **SSRF guard (`pkg/security.IsPrivateIp`) must be called before any user-supplied-URL dial** ‚Äî explicitly named as a consumer obligation for mcp, ai, knowledge, and workflows; not proven done anywhere yet. If MCP or wiki-api makes outbound calls driven by user input (e.g., fetching a URL a user pastes), this is a binding requirement, currently unverified everywhere.
- **Layering**: per user's own standing rule (no-fallbacks/no-legacy memory) and lib's A-CLIENTS discipline, satellites must go through the typed lib clients (engineclient/knowledgeclient/billingclient/identityclient), never direct-to-engine or hand-rolled sibling calls ‚Äî and sibling-service fakes in tests must come from contracts golden fixtures only (CS ¬ß5 Row 3), never hand-authored guesses. This directly binds how MCP/wiki-api/CLI test their engine and knowledge integrations.
- **Observability**: `pkg/obs.Setup` is currently a silently-succeeding no-op (FR-L31, a live defect) ‚Äî any surface currently "wiring obs" via lib gets nothing and a false green. The redo's architecture should not assume lib-provided OTel is live; must verify or state the dependency explicitly, and should track FR-L31's fix.
- **Contracts pin instability (MR-1) blocks correctly testing any lib-mediated sibling client** ‚Äî since wiki-api/mcp/cli either import lib clients or parallel their pattern, the redo's test-design should account for: which contracts tag is authoritative is unresolved family-wide as of 2026-07-15/16-17 mirror date; golden fixtures may not reflect the current dev-authoritative shape.
- **Idempotency-Key on `/internal/*`**: cell contract Rule 11 mandates this; lib is stated as the owner of the shared helper but it is NOT YET EVIDENCED (SDD ¬ß5, Rule 11 unticked). If MCP or wiki-api expose internal step-APIs, this is an owed dependency not yet available from lib.
- **No Linear anywhere** (D-S11, platform-wide) ‚Äî confirms existing memory; no MCP/CLI/API verb should reference Linear as a first-class integration via this shared lib (a residual `linear*` DfM block is only ever an opaque-preserve handle).
- **ADR-0035 boundary**: any redo PRD claiming "Go client codegen" for MCP/CLI/wiki-api from contracts must not cite ADR-0035 ‚Äî it is TS-only. Go clients (including anything the redo needs from lib) are and will stay hand-authored.
