## Reading rules (the traps this list is written against)

1. **Code-at-HEAD, not doc status.** `certified ≠ current`: several PRD/canon pages self-declare DRAFT while `status:canonical`. Every coverage call cites a `file:line`/route/ticket. The engine's amazing-MCP primitives are code-real but **"live end-to-end is deploy-pending"** (`engine §3`, commit `91b3b115`) — "done" here means real+wired at HEAD, not behaviorally verified in prod.
2. **Serving service is a first-class column (PO-D4).** A capability parked on the wrong service is a gap, not coverage. Legend: `wiki-api` = the Go `/v1` tier · `engine` = AGPL engine primitive fronted by `/v1` (SoR for writes+collab, **never the AI path**) · `knowledge` = orvex-studio-knowledge (search/related/RAG) · `ai` = orvex-studio-ai (ask/chat/inline/image/bake) · `identity` = orvex-studio-identity (authn/JWKS) · `audit` = orvex-studio-audit (event store+query, R16/R25 — **the ONLY owner of audit storage/WORM/read-model/query; every other service is EMIT-ONLY**) · `billing` = orvex-studio-billing (entitlements) · `contracts` = orvex-studio-contracts (the spine). "wiki-api-fronted" means `/v1` composes/transports to a satellite.
3. **Disposition is argued from first principles (PO-D2), never "the fork had it" or "the ticket had it":** `adopt` (carry as-is) · `adopt-reshaped` (carry, changed shape/home) · `rethink` (open design call, recommendation given) · `drop`.
4. **Two OBE ticket clusters.** The early ENG-13xx–19xx Done batch shipped the capability; the later ENG-24xx–25xx "from-scratch" Todo batch re-proposes it. Coverage cites the Done batch; §(b) flags the re-proposals.
5. **Atlas confirmed present** (`wiki-study/atlas.md`, 61KB, produced after the draft was written — draft's "atlas does not exist" note is now stale). Cross-checked for serving-home / routing bindings that touch this list; one live tension found and resolved (Appendix B).
